# 角色设计流程

角色设计专用流程。被 novel-discuss 识别为"角色设计"话题时加载执行。

## 目标

创建立体、有深度、令人印象深刻的角色，包括主角、配角和反派。

## 核心关注点

1. **辨识度** — 读者能记住
2. **记忆点** — 提起角色就想起的梗、画面、口头禅（参考《银魂》：银时的自然卷+死鱼眼+挖鼻孔+甜食党）
3. **功能定位** — 服务于剧情
4. **成长空间** — 有发展弧线

## 使用方式

```
/novel-discuss 创建角色              # 交互式创建
/novel-discuss 苏寒染                # 编辑/查看指定角色
/novel-discuss 设计一个反派          # 指定类型
```

## 执行流程

### 第一步：确定角色类型

使用 AskUserQuestion 展示选项：

| 类型 | 核心设计点 |
|------|-----------|
| 主角 | 核心性格、金手指、成长轨迹、核心动机 |
| 重要配角 | 与主角关系、功能定位、独立故事线、性格亮点 |
| 反派 | 与主角对立原因、层次感、魅力点、最终命运 |
| 工具人/路人 | 基本特征、出场目的 |

如用户已指定角色名，先检查是否已有设定文件（`memory/entities/characters/{名}.md`），有则进入编辑模式。

### 第二步：基础信息收集

根据角色类型收集信息。使用苏格拉底式提问，每个关键维度提供 2-3 个方案。

**主角/配角核心维度**：
- 姓名（可调用 novel-name 生成候选）
- 年龄（开局时）
- 开局身份
- 外貌特征（1-2 个显著特点）
- 核心性格（2-3 个关键词）

**反派额外维度**：
- 与主角对立原因
- 层次感设计（不是纯粹的坏）

### 第三步：记忆点设计（银魂法）

每个角色至少需要：1 个视觉记忆点 + 1 个行为记忆点 + 1 个可反复使用的梗。

```
记忆点类型（参考《银魂》经典角色）

1. **视觉记忆点** — 读者"看到"就想起
   定义：标志性的外观特征或随身物品
   示例：银时（自然卷+死鱼眼+蓝色和服）、神乐（大眼睛+暗红和服+伞）
   设计要点：1-2个足够但要反复出现；可被其他角色吐槽
   避免：写了一堆外貌描写但没有一个"记住这个"
   正确：选一个最极端的特征，其他全部服从它

2. **行为记忆点** — 读者"想到"就笑出
   定义：习惯性小动作或行为模式
   示例：银时（挖鼻孔、看Jump、赖床）、冲田（虐待土方、踩东西进门）
   设计要点：换个人做这件事就不对了；可以是反差萌；可被其他角色"预防"

3. **语言记忆点** — 读者"模仿"就上瘾
   定义：口头禅、标志性句式、特殊说话方式
   示例：银时（"喂喂"、"我不是xx，只是xx而已"）、神乐（"阿鲁"）
   设计要点：口头禅短，3-5字最佳；标志性句式可变形使用

4. **反差记忆点** — 读者"发现"就惊艳
   定义：与核心人设形成强烈矛盾的特质
   示例：银时（平时懒散→关键时刻帅到炸裂）、土方（铁面无私→写俳句）
   设计要点：反差越大越好但有内在逻辑；可成为伏笔

5. **梗/重复笑点** — 可反复使用，越用越熟
   定义：可反复触发的情境组合或人物互动模式
   示例：银时vs土方（互相嫌弃但关键时刻默契配合）
   设计要点：需两个以上角色互动才有化学反应；可进化
```

**记忆点检查**：
- 闭上眼能"看到"这个角色吗？（视觉）
- 模仿这个角色说话容易吗？（语言）
- 这个角色有个"只有TA才会做的事"吗？（行为）
- 提起这个角色，读者第一个想到什么？（核心记忆点）

### 第四步：金手指设计（主角专用）

使用 AskUserQuestion 提供 3-4 个方案，常见类型：
- 系统流（任务/签到/商城）
- 传承流（远古灵魂/残缺记忆）
- 体质流（特殊灵根/混沌体）
- 重生/穿越流（前世记忆/现代知识）

金手指设计原则：有明确来源和限制、与成长形成正反馈、不破坏悬念。

### 第五步：人物关系设计

读取已有角色列表，设计新角色与现有角色的关系。如有需要，调用 novel-name 生成新关系人的名字。

### 第六步：行为底层逻辑

为每个核心角色设定一个"不妥协"的执念，决定他们在面临选择时的核心动机。

```
问自己：这个角色在所有情况下都不会放弃的是什么？

苏寒染：不会放弃任何需要帮忙的人（即使嘴上吐槽）
凌绝尘：不会让同伴替他挡刀
反派：不会为了目标牺牲自己的尊严

注意：行为底层逻辑 ≠ 表面目标
表面目标可以变，底层逻辑不变。
```

### 第七步：生成角色卡片

输出到 `memory/entities/characters/{角色名}.md`，格式：

```markdown
## {角色名}

> 角色类型：{主角/配角/反派}
> 创建时间：{日期}
> 最后更新：{日期}

### 基本信息
- **姓名**：{名}
- **年龄**：{年龄}（开局时）
- **身份**：{开局身份}
- **境界**：{初始境界}
- **灵根/特殊体质**：{如有}

### 外貌特征
- {1-2个显著特征，记忆点优先}

### 核心性格
- **{性格1}**：{简述}
- **{性格2}**：{简述}

### 性格标签
- 优点：{列表}
- 缺点：{列表}

### 金手指（主角专用）
{描述，含限制条件}

### 动机与目标
- **表面目标**：{短期目标}
- **深层目标**：{隐藏动机}
- **核心动机**：{行为底层逻辑}

### 记忆点
- 视觉：{特征}
- 行为：{习惯}
- 语言：{口头禅/句式}
- 反差：{矛盾特质}
- 梗：{互动模式}

### 人物关系
- {角色名} → {关系}：{描述}

### 当前状态（随写作进度更新）
- 位置：{当前位置}
- 境界：{当前境界}
- 已知信息：{角色当前知道的}

### 背景故事
{故事开始前的经历}
```

## 内容边界规则

角色设定文件只记录"已确定的内容"，不记录"未来规划"。

**允许写入**：
- 初始状态（故事开始时的身份、境界、处境）
- 背景故事（故事开始前的经历）
- 基础属性（年龄、外貌、性格、能力）
- 初始关系（故事开始时已建立的）
- 已完成成长（已写完章节后的变化）

**禁止写入**：
- 未来剧情发展 → 放章节大纲或 `memory/future/40-arcs/` / `memory/future/30-volumes/`
- 成长计划 → 放 `memory/future/30-volumes/` 或 `memory/future/10-book.md`
- 预期关系变化 → 放章节大纲或 `memory/future/40-arcs/`

**判断标准**：内容是否已经发生（章节已写完）？避免"计划"、"将"、"会"等未来时态。

## 角色设计原则

1. **差异化** — 每个角色有独特的标签，读者能一眼区分
2. **记忆点（核心）** — 至少 3 个记忆点（视觉+行为/语言+梗），没有记忆点的角色 = NPC
3. **功能性** — 服务于剧情（推动剧情/提供信息/制造冲突/情感支撑）
4. **一致性** — 行为符合性格设定，避免 OOC
5. **成长性** — 主要角色有成长弧线：起点状态 → 触发事件 → 内在转变 → 新的状态
6. **立场大于善恶** — 弱化善恶二分法，强化立场冲突。角色做事不是因为"坏"，而是立场与主角冲突

人物弧光原则：
- 弧光不等于"变好"，也可以是"变复杂"或"变极端"
- 每个弧光需要一个具体的触发事件
- 弧光必须有代价——成长不应该是免费的
- 主角必须有性格上的缺点或能力短板
