# 物品与道具设计流程

物品/法宝/道具/工具/阵法等实体设计专用流程。被 novel-discuss 识别为"物品设计"话题时加载执行。

适用于任何类型的物品：武器、防具、消耗品、载具、阵法、道具、科技产品、法宝等。

## 目标

设计有功能感、有辨识度、与世界观自洽的物品，确保每个物品在剧情中有明确的叙事功能。

## 核心关注点

1. **功能性** — 物品能做什么？限制是什么？读者能理解规则
2. **辨识度** — 读者能记住这个物品（独特外观、特殊使用方式、有趣限制）
3. **叙事功能** — 物品在剧情中扮演什么角色？（推进剧情/制造冲突/揭示信息/提供笑点）
4. **平衡性** — 不过强破坏悬念，不过弱没有存在感

## 使用方式

```
/novel-discuss 设计一把剑          # 具体物品
/novel-discuss 阵法怎么设计          # 某类物品的方法论
/novel-discuss 金手指设计             # 特殊物品
/novel-discuss 经济体系               # 物品体系级讨论
```

## 执行流程

### 第一步：读取现有设定

调用 novel-knowledge 搜索已有物品和体系：
```
skill="novel-knowledge", args="search 物品 --type item"
skill="novel-knowledge", args="search 阵法 --type concept"
```

同时检查：
- `memory/worldbuilding.md` — 物品体系总览（分级标准、经济体系等）
- `memory/entities/items/` — 已有物品
- `memory/entities/concepts/` — 相关体系概念（如阵法体系、功法体系）

### 第二步：确定物品类型

| 类型 | 核心设计点 | 示例 |
|------|-----------|------|
| 武器/攻击型 | 伤害方式、限制条件、成长性 | 剑、枪、法杖、炸弹 |
| 防具/防御型 | 防护范围、弱点、代价 | 盾、护甲、结界、屏障 |
| 工具/功能型 | 核心功能、使用条件、日常感 | 通讯器、照明灯、净化器 |
| 消耗品 | 效果、稀有度、副作用 | 丹药、卷轴、符箓 |
| 载具/移动型 | 速度、载量、使用限制 | 飞剑、马车、飞船 |
| 特殊/剧情道具 | 独特性、来历、伏笔功能 | 信物、钥匙、圣物 |
| 体系级概念 | 规则框架、分级标准、边界 | 阵法体系、丹药体系、货币体系 |

如用户已指定物品名，先检查是否已有设定文件，有则进入编辑模式。

### 第三步：苏格拉底式设计

根据物品类型，选择对应的讨论路径。

#### 路径 A：单个物品设计

问题序列（每个问题提供 2-3 个方案）：

1. **核心功能**：这个物品解决什么问题？
   - 方案方向：战斗辅助 / 日常便利 / 情报获取 / 生存保障
2. **使用条件**：谁可以用？需要什么前提？
   - 方案方向：所有人可用 / 特定职业 / 特定境界 / 特殊体质
3. **限制与代价**：不能无限使用，代价是什么？
   - 方案方向：消耗资源 / 冷却时间 / 副作用 / 耐久损耗 / 使用者承受
4. **外观特征**：读者"看到"就能记住的设计
   - 方案方向：形状独特 / 材质特殊 / 伴随现象（发光/声音/气味）
5. **叙事功能**：它在剧情里制造什么？
   - 方案方向：危机转机 / 信息揭示 / 角色互动道具 / 喜感来源

#### 路径 B：物品体系设计

问题序列：

1. **分级标准**：物品如何分级？（几级？按什么分？）
   - 通用框架：材质/威力/稀有度/制造难度 — 至少选一个作为主轴
   - 经典模式：凡品 → 灵品 → 宝品 → 仙品（四级制，可调整级数）
   - 非修真项目：普通 → 稀有 → 史诗 → 传说（游戏化分级）
2. **经济循环**：物品怎么流通？
   - 获取途径：制造/交易/拾取/任务奖励
   - 货币锚定：什么物品是硬通货？
   - 供需关系：什么物品稀缺？什么过剩？
3. **与现有体系的关联**：和修炼/能力体系怎么挂钩？
   - 物品是否依赖使用者能力？
   - 物品是否能弥补能力不足？
   - 高阶物品的使用门槛？

#### 路径 C：阵法/技能体系设计（如项目有阵法/魔法阵等体系）

问题序列：

1. **核心原理**：这个体系的基本规则是什么？（用一句话说清楚）
2. **实现方式**：怎么部署/释放？
   - 方案方向：永久预置 / 临时绘制 / 消耗品触发 / 修炼技能
3. **分级规则**：
   - 几级？每级差异是什么？（复杂度/威力/材料/门槛）
   - 命名规范？（统一的命名方式让读者一看就知道等级）
4. **属性/元素绑定**：是否有属性限制？
   - 方案方向：严格绑定 / 部分绑定 / 无限制
5. **成长路径**：使用者如何从低级到高级？

### 第四步：平衡性检查

设计完成后，自动检查：

- **强度检查**：这个物品在当前等级/阶段是否过强？是否让主角没有成长空间？
- **唯一性检查**：是否和已有物品功能重叠？如果有，差异在哪里？
- **限制充分性**：代价是否足以防止滥用？
- **剧情适配**：这个物品是否能为剧情创造有趣的使用场景（而不仅仅是"更强了"）？
- **信息可理解性**：读者能否在不回头查设定的情况下理解这个物品的规则？

### 第五步：生成物品设定文件

输出到 `memory/entities/items/{物品名}.md`，格式：

```markdown
---
id: item_xxx
name: 物品名称
type: item
tags:
  - 类型标签
  - 来源标签
relations:
  - target: 相关实体id
    relation: 关系类型
    description: 关系描述
---

# 物品名称

## 基本信息
- **性质**：{一句话定义}
- **类型**：{武器/防具/工具/消耗品/载具/特殊}
- **品阶**：{等级}
- **来源**：{谁制造的/从哪来的}

## 外观与规格
- **形状**：{描述}
- **材质**：{描述}
- **特征**：{读者能记住的外观特点}

## 功能
- **核心功能**：{主要作用}
- **使用方式**：{怎么用}
- **使用条件**：{谁能用/需要什么前提}

## 限制与代价
- {限制1}
- {限制2}

## 数值参数（如适用）
- {威力/范围/持续时间/数量等具体数值}

## 在剧情中的作用
- {叙事功能}
- {可能的剧情用法}
```

**体系级设定**（如阵法体系、丹药体系等）应写入 `memory/entities/concepts/{体系名}.md`，格式参见 concept-design 流程。

### 第六步：同步到知识图谱

```
skill="novel-knowledge", args="update item {物品名} --content {描述}"
skill="novel-knowledge", args="rebuild"
```

## 物品设计原则

1. **规则先行** — 先定规则再定数值，规则比数值更重要
2. **限制即特色** — 最好的物品设计，限制条件本身就是记忆点
3. **叙事驱动** — 物品存在的根本原因是为剧情服务，不是为体系服务
4. **读者可理解** — 一个物品的规则不应该需要"查说明书"
5. **成长空间** — 给物品留出升级/进阶/被替代的可能性
6. **独特辨识** — 每个重要物品至少有一个"只有它才有"的特征

## 内容边界规则

只记录已确定的设定，不记录未来规划。

**允许写入**：
- 物品的固有属性和规则
- 已知的制造方式和使用方法
- 与其他实体的关系

**禁止写入**：
- 未来剧情中的使用方式 → 放章节大纲或 `memory/future/40-arcs/`
- 计划中的升级路径 → 放 `memory/future/30-volumes/` 或 `memory/future/40-arcs/`
- "可能"出现的变体 → 放 `memory/future/` 对应规划文件
