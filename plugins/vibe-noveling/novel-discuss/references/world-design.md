# 世界观引导式构建

从零引导构建完整小说世界观的流程。支持两种模式：
- **引导模式**：按模块逐个推进，进度持久化到文件，跨 session 续接
- **自由模式**：针对具体世界觀话题进行讨论

被 novel-discuss 识别为"世界观"相关话题时加载。

## 使用方式

```
/novel-discuss 构建世界观           # 引导模式（读取进度文件）
/novel-discuss 从零设计设定         # 引导模式
/novel-discuss 世界观进度           # 查看进度
/novel-discuss 讨论一下地理         # 自由模式（直接讨论地理）
/novel-discuss 历史背景怎么设       # 自由模式
```

---

## 第一步：判断模式

读取 `memory/world-design-progress.md`：

- **文件不存在 + 用户说"构建/设计世界观"** → 引导模式，先创建进度文件
- **文件存在 + 用户说"构建/设计世界观"或查看进度** → 引导模式，找到下一个未完成模块
- **用户指定具体话题**（如"地理设定"、"历史背景"） → 自由模式，跳到对应模块

---

## 第二步：引导模式 — 八大模块

每个模块解决一个核心问题。模块 1 是所有其他模块的基础，应最先讨论。其余模块顺序灵活，可按需跳转。

| # | 模块 | 核心问题 | 讨论工具 |
|---|------|---------|---------|
| 1 | 世界本质 | 这是什么样的世界？凭什么吸引人？ | 本文件内联 |
| 2 | 能力体系 | 力量/魔法/技术如何运作？ | concept-design.md |
| 3 | 社会形态 | 人们怎样生活？谁说了算？ | 本文件内联 |
| 4 | 势力格局 | 谁掌握权力？他们的矛盾是什么？ | faction-design.md |
| 5 | 地理环境 | 故事发生在哪里？ | 本文件内联 |
| 6 | 历史背景 | 什么塑造了现在的世界？ | 本文件内联 |
| 7 | 经济与物品 | 资源如何流动？有什么关键物品？ | item-design.md |
| 8 | 核心矛盾 | 什么推动故事？ | 本文件内联 |

**模块路由**：有专用 reference 的模块（2/4/7），讨论时读取对应文件按其流程执行，讨论完回到本文件更新进度。其余模块使用下方的内联框架。

### 引导模式执行流程

1. 展示当前进度：已完成 ✅ / 进行中 🔄 / 待设计 ⬜
2. 如果有进行中的模块，继续该模块
3. 如果没有，推荐下一个待设计模块（但允许用户选其他）
4. 进行苏格拉底式讨论
5. 讨论结束后更新进度文件和对应 entity 文件
6. 展示更新后的进度总览，询问是否继续下一个模块

## 第三步：内联讨论框架（模块 1/3/5/6/8 使用）

以下模块没有专用 reference，使用本文件的讨论流程。框架与 novel-discuss 的苏格拉底式方法论一致：每个问题提供 2-3 个差异化方案，尊重用户选择。

### 模块 1：世界本质

**目标**：用一句话定义这个世界，确定独特性和基调。

问题序列：
1. **类型定位**：这个世界的本质是什么？
   - 方案方向：现实延伸（现实世界 + 一个关键变量）/ 完全架空（自建规则）/ 历史改写（真实历史 + 转折点）/ 未来推演（从当前趋势外推）/ 平行世界（与我们相似但有本质差异）
2. **独特性**：和同类作品相比，最大的差异点是什么？
   - 原则：至少要有一个让读者觉得"没见过"的设定。不要小改动，要核心机制级的差异
3. **基调**：这个世界给人的整体感觉？
   - 方案方向：黑暗残酷 / 明朗热血 / 灰色现实 / 荒诞幽默 / 史诗宏大 / 温馨治愈

**输出**：写入 `memory/worldbuilding.md` 概述部分。

### 模块 3：社会形态

**目标**：确定社会如何组织，普通人怎么生活。

问题序列：
1. **政治体制**：这个世界由谁统治？
   - 方案方向：帝制/王权 / 民主/议会 / 神权 / 军阀割据 / 部落联盟 / 无政府/弱政府 / 公司/财阀统治
2. **阶层分化**：人分几等？依据什么？
   - 方案方向：按能力（修炼等级/天赋）/ 按血统（家族/种族）/ 按财富 / 按知识 / 阶层固化 vs 可流动
3. **普通人的生活**：大多数人的日常是什么样的？
   - 方案方向：安居乐业 / 被压迫剥削 / 在夹缝中求存 / 信仰驱动 / 追求向上流动
4. **文化与价值观**：这个世界推崇什么？鄙视什么？

**输出**：写入 `memory/worldbuilding.md` 社会结构部分。

### 模块 5：地理环境

**目标**：确定物理世界的基本面貌和关键地点。

问题序列：
1. **世界格局**：世界的整体形态？
   - 方案方向：大陆型（一块大陆为主）/ 群岛型（多块陆地分散）/ 星球型（多星球）/ 线型（沿河流/道路/山脉）/ 空间型（多层空间/位面）
2. **关键地点**：故事需要哪些场景？
   - 每个地点至少需要：名称、特征、叙事功能（起点/转折/高潮/终点）
   - 建议 3-7 个核心地点，不必一次定完
3. **特殊地形**：有没有影响剧情的地理元素？
   - 方案方向：天然屏障（不可跨越的障碍）/ 危险区域（禁地/荒漠）/ 资源富集区（争夺焦点）/ 交通枢纽（必经之地）
4. **地理与势力的关系**：地理如何影响势力分布？

**输出**：关键地点创建为 `memory/entities/locations/` 文件，总览写入 `memory/worldbuilding.md`。

### 模块 6：历史背景

**目标**：确定塑造当前世界格局的关键历史事件。

问题序列：
1. **起源**：这个世界/文明怎么来的？
   - 方案方向：神话起源（神创/远古大战）/ 历史演变（自然发展）/ 外来影响（移民/入侵/天灾）/ 不明（失落的过去，可做伏笔）
2. **转折事件**：什么重大事件改变了世界走向？（1-3 个）
   - 每个事件：时间、原因、结果、对现在的影响
3. **历史与主角**：主角/主角家族与历史有什么关联？
   - 方案方向：事件的直接参与者 / 受害者后代 / 无关但被卷入 / 发现历史真相的人 / 历史的创造者
4. **隐藏的历史**：有什么只有少数人知道的历史秘密？（为伏笔预留）

**输出**：写入 `memory/worldbuilding.md` 历史部分，大事件写入 `memory/past.md`。

### 模块 8：核心矛盾

**目标**：确定驱动故事的核心张力。在完成前面模块后讨论效果最佳。

问题序列：
1. **主要矛盾**：这个世界最大的冲突是什么？
   - 方案方向：阶级冲突（压迫 vs 反抗）/ 文明冲突（共存 vs 灭绝）/ 生存危机（外部威胁）/ 理念之争（不同的"正确"）/ 个人与体制
2. **矛盾层次**：矛盾在不同层面如何体现？
   - 宏观（世界级：文明存亡、体制危机）/ 中观（势力间：博弈、联盟）/ 微观（个人：命运与信念的碰撞）
3. **矛盾与主角**：主角如何卷入？
   - 方案方向：矛盾的中心 / 意外变量 / 调和者 / 被迫选边
4. **矛盾的演变**：矛盾会怎么发展？
   - 方案方向：逐步升级 / 反转（敌友转换）/ 暴露真相后重新定义 / 引出更大的矛盾

**输出**：写入 `memory/future/10-book.md` 的全书定位 / 困境升级部分；若形成长期冲突线程，再补到 `memory/future/20-threads.md`。

## 第四步：更新进度文件

每个模块讨论完成后，更新 `memory/world-design-progress.md`：

1. 将清单中的 `[ ]` 改为 `[x]`，一行简要说明
2. 在设计记录区填写讨论日期、关键决定（3-5 条）、待补充项

如果讨论到一半用户中断，标记为 `[🔄]` 进行中，记录已讨论的内容。

## 第五步：同步知识图谱

讨论产生的实体需要同步到知识图谱：

```
skill="novel-knowledge", args="update {type} {name} --content {描述}"
skill="novel-knowledge", args="rebuild"
```

各模块产出的实体类型：
- 模块 1 → `memory/worldbuilding.md`（无 entity 文件）
- 模块 2 → `memory/entities/concepts/`
- 模块 3 → `memory/entities/concepts/` 或 `memory/worldbuilding.md`
- 模块 4 → `memory/entities/factions/`
- 模块 5 → `memory/entities/locations/`
- 模块 6 → `memory/worldbuilding.md` + `memory/past.md`
- 模块 7 → `memory/entities/items/` + `memory/entities/concepts/`
- 模块 8 → `memory/future/10-book.md`（必要时补 `memory/future/20-threads.md`）

## 设计原则

1. **模块 1 优先** — 世界本质是其他一切的基础，应最先讨论
2. **模块 8 靠后** — 核心矛盾依赖前面模块的积累，放最后讨论效果最好
3. **顺序灵活** — 模块 2-7 可根据创作需要调整顺序或跳过
4. **渐进完善** — 先搭骨架再填血肉，不必一次设计完所有细节
5. **与 entity 文件联动** — 讨论结果最终落地为 entity 文件，进度文件只是索引
6. **允许回溯** — 后续讨论可能推翻前面的决定，更新进度文件记录变更即可

## 内容边界

- 进度文件只记录已讨论过的模块状态和关键决定摘要
- 详细设定写入 entity 文件或 worldbuilding.md
- 不编造未来规划，不做超出已讨论范围的假设
