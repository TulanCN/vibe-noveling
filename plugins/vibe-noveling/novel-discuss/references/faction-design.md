# 势力与组织设计流程

势力/组织/门派/帮派/阵营/国家等实体设计专用流程。被 novel-discuss 识别为"势力设计"话题时加载执行。

适用于任何类型的组织：修真门派、现代企业、军事组织、政治派系、种族势力等。

## 目标

设计有独特文化、内部矛盾、合理利益驱动、与剧情深度绑定的势力。

## 核心关注点

1. **独特性** — 每个势力有不可替代的特色（不是换个名字就一样）
2. **合理性** — 势力的行为由利益驱动，不是善恶驱动
3. **复杂性** — 内部有派系和矛盾，不是铁板一块
4. **剧情功能** — 势力为冲突提供舞台，为角色提供归属

## 使用方式

```
/novel-discuss 设计一个门派           # 具体势力
/novel-discuss 势力设定               # 交互式选择
/novel-discuss 这个组织的内部结构       # 补充已有势力
/novel-discuss 两方势力为什么开战      # 关系设计
```

## 执行流程

### 第一步：读取现有设定

调用 novel-knowledge 搜索已有势力和关系：
```
skill="novel-knowledge", args="search 势力 --type faction"
```

同时检查：
- `memory/worldbuilding.md` — 政治格局和社会形态
- `memory/entities/factions/` — 已有势力
- `memory/entities/characters/` — 已有角色（势力需要人）

### 第二步：确定讨论方向

| 方向 | 核心内容 | 适用场景 |
|------|---------|---------|
| 新建势力 | 从零设计一个完整的势力 | 项目初期/需要新势力出场 |
| 补充势力 | 为已有势力补充细节 | "待补充"状态的势力 |
| 关系设计 | 设计势力之间的关系 | 需要冲突/同盟/外交剧情 |
| 内部结构 | 设计势力内部的组织架构 | 需要角色在其中活动的细节 |

### 第三步：苏格拉底式设计

#### 路径 A：新建势力

问题序列（每个问题提供 2-3 个方案）：

1. **核心定位**：这个势力为什么存在？
   - 方案方向：利益共同体（商业/资源）/ 信仰组织（宗教/理念）/ 血缘纽带（家族/种族）/ 权力机构（政府/军事）
2. **核心资源/优势**：靠什么维持生存和扩张？
   - 方案方向：垄断资源 / 掌握关键技术 / 军事优势 / 情报网络 / 地理优势 / 人口规模
3. **组织结构**：内部怎么管？
   - 方案方向：集权制（一人说了算） / 议会制（多人博弈） / 等级制（晋升体系） / 松散联盟（利益绑定）
4. **核心人物链**：谁说了算？下面谁管事？
   - 方案方向：明主+能臣 / 双头制（军政分离） / 老大+继承人矛盾 / 幕后操纵者
5. **内部矛盾**：表面团结下的裂缝是什么？
   - 方案方向：新老派系之争 / 改革派vs保守派 / 利益分配不均 / 理念分歧 / 继承人危机
6. **与主角的关系**：主角和这个势力是什么关系？
   - 方案方向：归属（主角是其中一员） / 合作（利益交换） / 对立（核心冲突） / 潜入（卧底/探秘）
7. **文化特色**：有什么独特的文化或行为习惯？
   - 方案方向：独特传统 / 特殊礼仪 / 标志性服饰或符号 / 独有的训练/选拔方式

#### 路径 B：关系设计

问题序列：

1. **关系性质**：这两个势力之间是什么关系？
   - 方案方向：同盟 / 敌对 / 竞争 / 依附 / 表面和平暗中博弈
2. **利益驱动**：为什么是这种关系？各自图什么？
   - 核心原则：**没有永远的朋友/敌人，只有永远的利益**
3. **历史根源**：这种关系是怎么形成的？
   - 方案方向：历史恩怨 / 资源争夺 / 理念冲突 / 外部威胁逼和
4. **破裂/变化点**：什么情况会让关系改变？
   - 为剧情预留关系转折的可能性

#### 路径 C：补充已有势力

直接读取已有设定文件，对比以下检查清单，逐项确认是否需要补充：

**势力深度检查清单**：

| 维度 | 检查项 | 优先级 |
|------|--------|-------|
| 领导层 | 掌权者是谁？决策风格？ | 高 |
| 核心利益 | 这个势力最在乎什么？ | 高 |
| 内部派系 | 有没有暗流？ | 高 |
| 军事力量 | 战斗力如何？靠什么打？ | 中 |
| 经济基础 | 靠什么赚钱/维持运转？ | 中 |
| 独有资源 | 有什么是别人没有的？ | 中 |
| 文化特色 | 有辨识度的传统/规矩？ | 低 |
| 外部关系 | 和其他势力什么关系？ | 高 |
| 与主角关系 | 主角怎么卷入？ | 高 |

### 第四步：势力一致性检查

- 这个势力的实力是否与地位匹配？（强到没对手 = 没有冲突可写）
- 内部矛盾是否可能被外部利用？（给主角操作空间）
- 势力的行为逻辑是否自洽？（如果A和B有矛盾，为什么还会合作？）
- 势力数量是否过多？（5-7 个核心势力足够，太多读者记不住）
- 是否有势力的功能重叠？（两个势力做的事一样 → 合并或区分）

### 第五步：生成势力设定文件

输出到 `memory/entities/factions/{势力名}.md`，格式：

```markdown
---
id: faction_xxx
name: 势力名称
type: faction
tags:
  - 类型标签
  - 阵营标签
relations:
  - target: 其他实体id
    relation: 关系类型
    description: 关系描述
---

# 势力名称

## 基本信息
- **性质**：{一句话定义}
- **位置/地盘**：{在哪里}
- **定位**：{在格局中的位置}

## 势力特点
- {1-3 个最核心的特点}

## 组织结构
- **权力架构**：{谁说了算，下面谁管事}
- **核心人物**：
  - {姓名} — {职位}：{简述}
- **内部派系**：{如有}

## 核心利益
- {最在乎的事}
- {次要利益}

## 力量
- **军事力量**：{如有}
- **经济基础**：{靠什么维持}
- **独有资源/技术**：{别人没有的东西}

## 文化特色
- {独特的传统、规矩、符号}

## 与各方势力关系
- **{势力名}**：{关系} — {原因}

## 待补充
- [ ] {需要补充的内容}
```

### 第六步：同步到知识图谱

```
skill="novel-knowledge", args="update faction {势力名} --content {描述}"
skill="novel-knowledge", args="rebuild"
```

## 势力设计原则

1. **立场大于善恶** — 势力做事不是因为"好"或"坏"，是因为利益和立场
2. **内部有裂缝** — 铁板一块的势力没有故事可写，内部矛盾给角色操作空间
3. **人有面目** — 核心人物要有至少一个辨识度特征，不要只是"某某长老"
4. **利益明确** — 每个势力的核心利益必须一句话说清楚
5. **可变化** — 势力的关系和立场应该可以随剧情变化，不要写死
6. **克制数量** — 核心势力 5-7 个足够，多了读者记不住

## 内容边界规则

同 character.md 和 item-design.md 的规则：只记录已确定的内容，未来规划放 `memory/future/` 对应文件。
