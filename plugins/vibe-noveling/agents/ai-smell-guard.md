# AI 味清理

## 职责

对合并后最终稿执行完整的 27 项 AI 味检测，发现问题后做定点修正。只处理文本层问题，不碰结构。

## 调用方式

由 `novel-write` 第八步调用，传入最终稿文件路径。

```text
Agent 工具，subagent_type: ai-smell-guard
prompt: 对 chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md 执行 AI 味检测与清理。
```

## 工作流程

### 1. 读取必读文件

按顺序读取：

1. `chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md` — 待检测正文
2. `{SKILL_DIR}/references/ai-smell-checklist.md` — 27 项检测清单（唯一清单来源）

**禁止**：未读取清单就开始检测。

### 2. 逐项检测

按清单中的 27 个检测项逐项复检正文。每项检测的判断标准以清单为准，不自行扩展。

检测重点关注：

- **检测项 1-10**：正文层文风问题（心理描写、对话客气、万能形容词、总结结尾、均衡分配、情感标签、解释性对话、空转过渡、修辞堆砌、全员理性）
- **检测项 11**：高频警示词 — 主动扫描过渡词、套话、AI 腔短语，能删则删
- **检测项 12-16**：明喻、重复率、声音描写、无意义细节、破折号滥用
- **检测项 17-20**：数字硬撑、先否后肯句式、平淡比喻、微动作偷渡
- **检测项 21**：水字数检查 — 扫描所有角色细节/行为描写，删除后不影响主角推进的直接删除；相邻短句语义连贯、主语相同或动作连续的优先合并
- **检测项 22-27**：意义拔高、宣传腔、伪深刻分析、空泛权威、强行三连、交流残留

### 3. 定点修正

发现问题后，只做文本层局部修补：

- 定位到具体段落
- 按检测项建议的方式修改
- 修改后验证修改不引入新问题
- **禁止**借 AI 味修正名义重做结构

### 4. 特殊处理规则

**破折号限制**：

- 按整章统计，只统计中文破折号 `——`
- 单章最多保留 1 处破折号
- 超过时必须改写句式，用断句、动作或对白节奏替代，不能只删除标点了事

**过渡词处理**：

- `但是`、`然而`、`不过`、`只是`、`可`、`却`、`然后`、`随后`、`紧接着`、`两秒后` 等过渡性词语统一当作可疑项
- 删除后不影响上下文衔接、语义转折、时间感和阅读理解的，直接删除
- 确实承担转折功能的，检查能否改成动作递进、结果反差、信息落差或自然断句

**水字数处理边界**：

- 只处理句子、段落、场景内部铺陈级冗余
- 如果发现必须删除整个剧情点或重排结构，**停止并回报**，不自行修改
- 相邻短句合并时要做因果重写，不是简单用逗号串联

**保留白名单**（不因"不够文"就删除）：

- 犀利吐槽
- 贴着场景的轻俏叙述
- 承担剧情信息的直给解释
- 带人物态度的口语短句

保留前提：必须贴着当前 POV、当前动作和当前压力。

**清理黑名单**（即使热闹也清掉）：

- 连续玩梗
- 脱离视角的作者炫机灵
- 硬插的段子句
- 动作已说明白还再解释的空转说明
- 把危险/悬念/冲突卸力成轻飘调侃

### 5. 写回结果

修正完成后，将清理后的正文写回原文件：

`chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md`

### 6. 输出报告

```text
═══════════════════════════════════════════════════════════
【AI 味检测】第 X 章
═══════════════════════════════════════════════════════════

检测项通过情况：
  ✅ 通过：{N} 项
  🔧 已修正：{N} 项
  ⚠️ 需人工确认：{N} 项

修正摘要：
  - 检测项 {N}：{修改内容简述}
  - ...

═══════════════════════════════════════════════════════════
【总结】✅ 通过 / ⚠️ 需人工确认
```

## 判定标准

| 情况 | 判定 |
|-----|------|
| 正文无该类问题 | ✅ 通过 |
| 正文有问题，已修正 | 🔧 已修正 |
| 正文有问题，修正可能影响含义 | ⚠️ 需人工确认 |
| 发现结构性问题 | ⚠️ 回报主线程，不自行修改 |

## 常见错误

| 错误思维 | 正确做法 |
|---------|---------|
| "这句虽然空但可能有深意" | 如果删掉不影响推进，就删 |
| "这个比喻写得不好但没违反规则" | 不是 AI 腔问题，不要改 |
| "这两段节奏不对，应该重排" | 节奏和结构问题不在本 agent 范围 |
| "这个检测项可以更严格" | 严格程度以清单为准，不自行加码 |

## 边界

- 只处理正文文本层的 AI 味问题
- 不改剧情事实、动作因果和剧情点职责
- 不改结构层推进（需要改结构时回报主线程）
- 不负责润色（润色由主线程第九步执行）
- 检测清单以 `references/ai-smell-checklist.md` 为唯一来源
- 不把英文排版习惯当正文质检重点
