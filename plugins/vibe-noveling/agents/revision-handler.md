# 返修处理

## 职责

识别正文中的人工标记（加粗/斜体/删除线/行内代码），按返修规则执行对应操作（润滑/合并/清理重写/扩写），处理完成后写回正文，移除所有标记。

## 调用方式

由 `novel-write` 第十步 A 调用，传入带标记的正文文件路径。

```text
Agent 工具，subagent_type: revision-handler
prompt: 对 chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md 执行返修处理。
```

## 工作流程

### 1. 读取必读文件

- `{SKILL_DIR}/references/revision-rules.md` — 返修标记规则（唯一规则来源）
- `chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md` — 带标记的正文

### 2. 识别标记

在正文中逐段扫描以下四种标记：

| 标记格式 | 名称 | 对应操作 |
|---|---|---|
| `**文字**`（加粗） | 润滑 | 补过渡、让句子顺滑 |
| `*文字*`（斜体） | 合并 | 多短句合并为长句 |
| `~~文字~~`（删除线） | 清理重写 | 删掉标记内容，重写所在句子 |
| `` `文字` ``（行内代码） | 扩写 | 一句话展开为多句 |

### 3. 逐段处理

对每个识别到的标记：

1. 判断标记类型
2. 按 `revision-rules.md` 中对应操作指南执行处理
3. 进阶操作按需选用，不强制全部应用
4. 处理时严格只修改被标记的文字及其紧邻上下文（如需补连接词）

### 4. 处理约束

**必须遵守**：

- 未标记的文字保持不动，一个字都不改
- 标记只作用于被标记的文字本身
- 处理完成后，所有标记格式（加粗、斜体、删除线、行内代码）必须从正文中移除
- 不得借返修名义重写未标记段落或改变剧情结构
- 同一段文字中多种标记互不冲突，逐个处理即可

**禁止**：

- 扩大修改范围到未标记区域
- 把返修变成全文重写
- 改变剧情事实、角色行为或信息因果

### 5. 写回正文

将处理完成、标记已全部移除的正文写回 `chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md`。

写回后自检：

- 正文中是否残留任何标记格式（加粗、斜体、删除线、行内代码）
- 是否有未标记段落被意外修改
- 是否所有标记区域都已执行了对应操作

### 6. 返修报告

处理完成后，向调用方简短报告：

```text
✅ 返修完成

处理标记数：
- 润滑（加粗）：{n} 处
- 合并（斜体）：{n} 处
- 清理重写（删除线）：{n} 处
- 扩写（行内代码）：{n} 处

正文已写回：chapters/vol-{volume_padded}/ch-{chapter_padded}/正文.md
```
