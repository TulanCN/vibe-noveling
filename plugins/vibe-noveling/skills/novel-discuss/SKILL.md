# 通用讨论框架（自由讨论话题使用）

角色设计和各类设定讨论有各自的专用流程（见 reference 文件）。自由讨论话题使用以下框架。

## 核心理念

苏格拉底式对话的本质：
1. **不直接给答案** — 帮助用户思考
2. **提供多角度** — 展示不同可能性
3. **尊重用户选择** — 最终决定权在用户

## 执行流程

### 1. 理解问题

```
你提到的问题是："{用户的问题}"

让我确认一下我的理解：
- 核心问题：{提炼}
- 背景：{从上下文提取}
- 限制条件：{从上下文提取}

理解正确吗？
```

### 2. 澄清性提问（如需要）

问题不够清晰时，先提问澄清，不要急着给方案。

### 3. 生成三个方案

使用 AskUserQuestion 工具展示结构化选项：

```
方案 A：【{名称}】
  描述：{详细描述}
  优点：{优点}
  缺点：{缺点}
  适合：{适合的情况}

方案 B：【{名称}】
  （同上）

方案 C：【{名称}】
  （同上）
```

### 4. 根据用户反馈迭代

- **用户选择方案** → 确认含义，记录决定
- **用户要求混合** → 逐项确认从各方案取什么
- **用户不满意** → 问清楚不满意的原因，重新设计
- **用户要更多细节** → 展开说明

### 5. 确认并记录

重要讨论必须按“单次讨论 = 单独文件”记录，不要再把完整报告直接写进同一个文件。

#### 记录位置

- 完整讨论报告：写入 `memory/discussions/`
- 讨论索引：追加摘要到 `memory/discussions.md`
- 若 `memory/discussions/` 不存在，先创建目录再写入

#### 文件命名规则

报告文件名格式：

`YYYY-MM-DD-HHMM-简短主题.md`

要求：

- 时间使用当前讨论完成时的本地时间
- `简短主题` 控制在 4 到 12 个字，概括本次讨论核心
- 删除文件名中的空格、斜杠、冒号及其他不适合路径的特殊字符
- 如果一时无法提炼主题，使用 `未命名话题` 作为兜底

示例：

- `memory/discussions/2026-04-05-1530-主角动机.md`
- `memory/discussions/2026-04-05-2215-宗门权力结构.md`

#### 完整报告模板

将完整内容写入 `memory/discussions/YYYY-MM-DD-HHMM-简短主题.md`：

```markdown
# {话题}

> 讨论时间：{YYYY-MM-DD HH:MM}
> 报告文件：`memory/discussions/{YYYY-MM-DD-HHMM-简短主题}.md`

## 问题
{问题}

## 背景与约束
- {背景1}
- {约束1}

## 选项
### 方案 A：{方案A名称}
{方案A描述}

### 方案 B：{方案B名称}
{方案B描述}

### 方案 C：{方案C名称}
{方案C描述}

## 最终选择
方案 {X} - {名称}

## 核心决定
- {决定1}
- {决定2}

## 后续行动
1. {行动1}
```

#### 索引追加格式

在 `memory/discussions.md` 末尾追加一条简短索引，便于后续统一检索：

```markdown
## {YYYY-MM-DD HH:MM} - {话题}

- 报告：`memory/discussions/{YYYY-MM-DD-HHMM-简短主题}.md`
- 结论：{一句话总结最终选择}
- 关键决定：{决定摘要}
```

## 方案设计原则

1. **差异化** — 三个方案核心思路明显不同，不是同一思路的微调
2. **完整性** — 每个方案都可独立执行
3. **客观性** — 优缺点分析客观，不预设倾向
4. **可行性** — 在当前项目背景下可行
