# 上下文路由

`novel-discuss` 的通用上下文读取策略。进入任何具体讨论流程前先读取本文件，再决定要读哪些项目文件。

## 核心规则

- 默认不全量扫描 `memory/`
- 只读取当前话题所需的最小相关上下文
- 先判定动作，再决定最小上下文包
- 对象优先于目录：已经知道要讨论谁、哪件物品、哪个势力时，优先读目标文件，不先扫整个实体目录
- 最近相关优先于历史全集：要查旧讨论时，先读 `memory/discussions.md` 和最近相关文件，不回溯全部历史
- 判断不清时，先追问对象或范围

## 第一步：判定动作

先判断这轮讨论属于哪一种动作，再决定上下文规模。

| 动作 | 典型信号 | 读取目标 |
|------|----------|----------|
| 新建设定 | “创建”“设计一个”“从零搭”“给我几个版本” | 为发散讨论准备最小背景，避免一开始就被历史文件淹没 |
| 补充已有设定 | “补一下”“细化”“把这个再展开” | 读取已有设定文件和直接相关文件 |
| 编辑已有实体 | 用户直接点名已有角色/势力/物品/概念，或说“改一下这个设定” | 优先读取目标实体，再补读必要关联 |
| 自洽性审查 | “审查”“检查矛盾”“是否冲突”“是否合理” | 读取目标设定和会发生冲突的少量交叉文件 |
| 未来剧情讨论 | “接下来怎么写”“这条线后面怎么办”“下一卷/下一 事件 怎么推进” | 只读取对应层级的 `future/` 文件和必要已发生剧情 |
| 自由讨论 | 通用方案比较、方向选择、创作决策 | 只读与当前问题直接相关的实体或最近讨论 |

## 第二步：按话题选择最小上下文包

下面的“先读”是起步包，“补读”只在确实命中相关时再追加。

### 世界观

**新建设定 / 引导构建**
- 先读：`memory/world-design-progress.md`、`memory/worldbuilding.md`（如存在）、`memory/setting-todo.md`
- 补读：只有当用户明确点名某个地点、势力、体系或历史事件时，再读取对应 entity 文件或相关 reference

**补充已有设定 / 编辑已有实体**
- 先读：`memory/worldbuilding.md`
- 补读：被点名的 `memory/entities/locations/`、`memory/entities/factions/`、`memory/entities/concepts/` 中的具体文件；必要时补最近相关讨论

**自洽性审查**
- 先读：`memory/worldbuilding.md` + 冲突涉及到的具体 entity 文件
- 补读：只有当矛盾明确牵涉未来规划时，才读相关 `memory/future/` 文件

### 角色

**新建设定**
- 先读：`memory/setting-todo.md`
- 补读：用户明确提到关系人、师门、宿敌、出身地时，再读取对应角色/势力/地点文件；必要时读 `memory/worldbuilding.md`

**编辑已有实体**
- 先读：目标角色文件 `memory/entities/characters/{角色名}.md`
- 补读：直接关系人、所属势力、相关物品；若这是延续上次讨论，再读最近相关讨论文件

**自洽性审查**
- 先读：目标角色文件 + 明确相关的势力/物品/概念文件
- 补读：只有当审查问题涉及已发生剧情或未来承诺时，才读 `memory/past.md` 或对应 `memory/future/` 层级

### 物品

**新建设定**
- 先读：`memory/worldbuilding.md`（如物品体系已存在）或 `memory/setting-todo.md`
- 补读：用户明确点名的相关概念、使用者、所属势力

**编辑已有实体**
- 先读：目标物品文件 `memory/entities/items/{物品名}.md`
- 补读：对应概念体系、制造者、持有者

**自洽性审查**
- 先读：目标物品文件 + 对应概念文件
- 补读：只有在冲突涉及既有剧情或未来规划时，才读 `memory/past.md` 或相关 `memory/future/`

### 势力

**新建设定**
- 先读：`memory/worldbuilding.md`
- 补读：用户明确提到的地盘、核心人物、对手势力；必要时读 `memory/setting-todo.md`

**编辑已有实体**
- 先读：目标势力文件 `memory/entities/factions/{势力名}.md`
- 补读：直接相关角色、地点、对手势力

**自洽性审查**
- 先读：目标势力文件 + 冲突涉及到的关系方
- 补读：只有在问题上升到卷、事件 或长期线程时，才读对应 `memory/future/` 文件

### 体系 / 概念

**新建设定 / 补充已有设定**
- 先读：`memory/worldbuilding.md`、目标概念文件（如存在）
- 补读：明确相关的角色、物品、势力文件

**编辑已有实体**
- 先读：目标概念文件 `memory/entities/concepts/{概念名}.md`
- 补读：直接依赖该体系的角色、物品、势力

**自洽性审查**
- 先读：目标概念文件 + 明确冲突对象
- 补读：只有当未来规划依赖这条规则时，才读对应 `memory/future/` 层级

### 自由讨论

**非未来剧情讨论**
- 先读：直接被点名的实体文件，或 `memory/discussions.md` 中最近相关条目
- 补读：只有当问题明确依赖某条世界规则、某段既有剧情或某个旧决定时，再追加对应文件

**未来剧情讨论**
- 先读：先判断讨论层级，不要默认同时读完整个 `future/`
- 全书级方向 / 终局 / 长期困境：`memory/future/10-book.md`
- 长期主线 / 伏笔 / 线程回收：`memory/future/20-threads.md`
- 某一卷的职责段 / 卷内推进：对应 `memory/future/30-volumes/vol-xx.md`
- 某个 事件 的阶段推进 / 中程对抗：对应 `memory/future/40-events/` 下的目标文件
- 补读：如果问题依赖已经发生的章节结果，再读 `memory/past.md`；如果这是延续某次未完成讨论，再读最近相关讨论文件

## 第三步：补读触发器

只有命中下面这些条件时，才继续扩大读取范围：

- 需要确认“之前已经怎么定过”时：读 `memory/discussions.md`，必要时再读最近相关报告
- 需要确认“这件事已经写出来没有”时：读 `memory/past.md`
- 需要确认“这个设定和世界底层规则是否冲突”时：读 `memory/worldbuilding.md`
- 需要确认“这个实体和谁绑定”时：读少量直接关联实体，不扫完整目录
- 需要把结论正式写进未来规划时：只读对应层级的 `memory/future/` 文件

## 第四步：什么时候停下来先问

出现下面情况时，不要靠扩大读取来赌答案，先追问：

- 不清楚用户是在新建、补充，还是修改已有设定
- 不清楚用户说的“这个角色 / 这个势力 / 这条线”具体指哪个对象
- 不清楚未来剧情讨论属于全书、卷、事件 还是单条线程
- 不清楚当前问题是在讨论设定，还是在讨论已发生剧情的复盘

原则：如果一个澄清问题能替代十个文件的盲读，优先提问。
