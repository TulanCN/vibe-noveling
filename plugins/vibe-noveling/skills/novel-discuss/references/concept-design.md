# 概念与体系设计流程

概念/体系/规则集等系统性设定设计专用流程。被 novel-discuss 识别为"概念/体系设计"话题时加载执行。

适用于任何类型的系统性设定：修炼体系、魔法体系、能力分级、经济规则、社会制度、技术原理等。

## 目标

设计内部自洽、边界清晰、与剧情互锁的规则体系，让读者在不知不觉中理解规则。

## 核心关注点

1. **一句话可说清** — 复杂体系的核心规则必须能用一句话概括
2. **内部自洽** — 体系内部没有自相矛盾
3. **与剧情互锁** — 体系规则能为剧情创造有趣的使用场景和限制
4. **渐进揭示** — 不需要一次交代完，分层释放给读者

## 使用方式

```
/novel-discuss 修炼体系怎么设          # 体系级设计
/novel-discuss 能力体系                 # 通用体系设计
/novel-discuss 这个体系的规则            # 补充/修改已有体系
/novel-discuss 自洽性检查              # 审查现有体系
```

## 执行流程

### 第一步：读取现有设定

调用 novel-knowledge 搜索已有体系和概念：
```
skill="novel-knowledge", args="search 体系 --type concept"
skill="novel-knowledge", args="search {关键词}"
```

同时检查：
- `memory/worldbuilding.md` — 现有体系总览
- `memory/entities/concepts/` — 已有概念体系
- `memory/entities/characters/` — 角色能力与体系的关联
- `memory/entities/items/` — 物品与体系的关联
- `memory/entities/factions/` — 势力与体系的关联

### 第二步：确定讨论方向

| 方向 | 核心内容 | 适用场景 |
|------|---------|---------|
| 新建体系 | 从零设计一个完整的规则体系 | 项目初期/需要新体系 |
| 补充体系 | 为已有体系补充规则细节 | "待设计"标记的子系统 |
| 修改体系 | 调整已有体系中的矛盾或不合理处 | 写作中发现设定冲突 |
| 审查体系 | 检查自洽性和完整性 | 大纲规划前/批量补充设定 |

### 第三步：苏格拉底式设计

#### 路径 A：新建体系

问题序列（每个问题提供 2-3 个方案）：

1. **核心本质**：这个体系用一句话怎么描述？
   - 方案方向：能量驱动型（修炼/蓄力） / 知识驱动型（掌握规则） / 血脉驱动型（天生能力） / 契约驱动型（交易/代价）
2. **层级/分级**：如何划分等级？
   - 方案方向：经典阶梯（线性升级） / 分支制（不同路线同级） / 突破型（质变节点） / 无级制（技能树式）
3. **核心限制**：什么阻止所有人变强？
   - 方案方向：天赋限制（资质/血脉） / 资源限制（材料/环境） / 时间限制（寿命/机会） / 认知限制（悟性/机缘）
4. **最强者**：这个体系的天花板在哪？为什么？
   - 方案方向：有天花板（被世界规则限制） / 理论无上限（但实际到不了） / 动态天花板（随世界变化）
5. **与日常生活的关联**：这个体系对普通人有什么影响？
   - 方案方向：完全隔离（修仙者vs凡人） / 部分渗透（货币/交通受影响） / 深度融合（人人相关）
6. **独特之处**：和同类作品中常见体系的差异点？
   - 核心原则：至少要有一个"别的作品没有"的机制

#### 路径 B：补充已有体系

读取已有体系文件，逐层检查完整性：

**体系完整性检查清单**：

| 维度 | 检查项 | 说明 |
|------|--------|------|
| 核心规则 | 一句话能说清吗？ | 读者需要理解的基础 |
| 分级 | 每级差异明确吗？ | 不能"感觉差不多" |
| 门槛 | 每级突破需要什么？ | 天赋/资源/机缘/时间 |
| 限制 | 这个体系不能做什么？ | 限制比能力更重要 |
| 交互 | 和其他体系怎么交互？ | 如修炼体系和物品体系 |
| 极端情况 | 边界条件是什么？ | 最强/最弱/最危险的情况 |
| 社会影响 | 这个体系改变了什么？ | 对社会结构/经济/政治的影响 |

#### 路径 C：审查体系（自洽性检查）

这是最关键的路径——当写作中发现设定矛盾时使用。

**自洽性检查框架**：

1. **内部一致性**
   - 规则之间是否有矛盾？（A规则说X，B规则说非X）
   - 分级标准是否一致？（不能有时按实力分，有时按资历分）
   - 限制条件是否合理？（太松=没有悬念，太紧=无法推进）

2. **外部一致性**
   - 与其他体系是否有冲突？（修炼体系 vs 物品体系的等级对应关系）
   - 与世界观是否匹配？（经济体系 vs 社会形态）
   - 与角色设定是否矛盾？（角色能力是否超出体系允许范围）

3. **剧情一致性**
   - 已写的章节中是否有违反体系规则的情况？
   - `memory/future/` 中的规划是否依赖不存在的体系规则？
   - 伏笔回收是否需要体系规则支持？

4. **强度/平衡检查**
   - 主角的能力是否在体系允许范围内？（太强=没有成长，太弱=不合理）
   - 反派的能力是否合理？（不能为了剧情需要突然变强/变弱）
   - 关键道具是否与体系兼容？

**检查输出格式**：

```
🔍 体系自洽性检查报告

【内部一致性】
  ✅ / ⚠️ / ❌ {发现}

【外部一致性】
  ✅ / ⚠️ / ❌ {发现}

【剧情一致性】
  ✅ / ⚠️ / ❌ {发现}

【强度平衡】
  ✅ / ⚠️ / ❌ {发现}

建议操作：
  A. 修改体系规则 → 调整 {具体规则}
  B. 修改角色/剧情 → 调整 {具体内容}
  C. 记录为已知限制 → 更新 {具体文件}
```

### 第四步：生成概念设定文件

输出到 `memory/entities/concepts/{概念名}.md`，格式：

```markdown
---
id: concept_xxx
name: 概念名称
type: concept
tags:
  - 体系标签
  - 分类标签
relations:
  - target: 相关实体id
    relation: 关系类型
    description: 关系描述
---

# 概念名称

## 基本信息
- **本质**：{一句话定义}
- **地位**：{在世界观中的位置}
- **适用范围**：{谁能用/哪里适用}

## 核心规则
{最基本的规则，用最简单的语言说清楚}

## 分级/层级
{如适用，描述等级划分和每级差异}

## 限制条件
- {限制1}
- {限制2}

## 子系统/分支
### {子系统名}
- {描述}

## 与其他体系的关系
- {与XX体系的关系}

## 待设计/待补充
- [ ] {需要补充的内容}
```

### 第五步：同步到知识图谱

```
skill="novel-knowledge", args="update concept {概念名} --content {描述}"
skill="novel-knowledge", args="rebuild"
```

## 体系设计原则

1. **简单核心** — 体系的核心规则一句话能说清，复杂度放在层级和细节里
2. **限制即有趣** — 体系最有意思的地方不是"能做什么"，而是"不能做什么"
3. **渐进揭示** — 不需要一开始就把所有规则告诉读者，随剧情需要分层释放
4. **规则服务于剧情** — 如果一条规则在剧情中永远不会被用到，考虑删掉
5. **独特性** — 至少要有一个区别于同类作品的独特机制
6. **可进化** — 为体系留出后续扩展的空间（新等级、新分支、新发现）

## 内容边界规则

同其他 reference 文件：只记录已确定的规则，未来规划放 `memory/future/` 对应文件。

**特别注意**：
- "待设计"的子系统用 `[ ] {描述}` 标记，不编造内容
- 数值参数（如"XX级需要YY年修炼"）尽量精确，模糊参数在写作时会造成矛盾
- 规则一旦写入正文，修改成本极高——设计阶段多花时间，写作阶段少踩坑
