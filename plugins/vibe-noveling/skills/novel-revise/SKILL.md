# 章节返修

## 何时使用

- 章节正文已交付，用户已在正文中加入标记
- 需要对标记的文字进行扩写、简写或润色

## 调用边界

- 只处理正文中被标记的文字及其紧邻上下文
- 未标记的文字一个字都不改
- 不负责正文创作，也不替代结构规划
- 如果返修过程中发现需要新增关键剧情、删除核心推进或重排结构，停止并回到 `novel-plan --revise`

## 目标

为正文中的每个标记生成 3 种不同策略的改写候选，由用户挑选最满意的版本。挑选后立即写入文件，处理完所有标记后提示用户进行同步。

## 核心理念

人最不可取代的部分是品味。AI 生成所有候选策略，人来判断用哪个。SSoT（String Seed of Thought）负责确保每个候选版本写法不同，避免趋同。

## 使用方式

```bash
/novel-revise 01
/novel-revise next
```

## 标记约定

| 标记 | 名称 | 含义 |
|---|---|---|
| `**加粗**` | 扩写 | 一句话太干，想多写点 |
| `~~删除线~~` | 简写 | 太啰嗦，需要压缩 |
| `*斜体*` | 润色 | 句子不顺/散/卡，需要调整 |

## 完整流程

```text
第一步：定位章节
     │  读取指定章节的 正文.md
     ▼
第二步：扫描标记
     │  识别所有标记（加粗/删除线/斜体）
     │  按出现顺序编号
     ▼
第三步：逐个处理标记（循环）
     │  对每个标记：
     │  ├─ 判断标记类型
     │  ├─ 用 SSoT 生成 3 种不同策略的改写候选
     │  ├─ 展示原文 + 3 个候选版本
     │  ├─ 用户选择（A/B/C 或自定义）
     │  └─ 选择后立即写入文件，移除标记格式
     ▼
第四步：交付与提示
     │  所有标记处理完毕
     │  提示用户运行 /novel-sync
     ▼
  ✅ 完成
```

## 第一步：定位章节

根据参数定位章节目录：

- `/novel-revise 01` → `chapters/vol-{volume_padded}/ch-0001/正文.md`
- `/novel-revise next` → 读取最新章节

读取 `正文.md` 全文。

## 第二步：扫描标记

扫描正文中所有标记，按出现顺序编号。

标记识别规则：

- `**加粗**` → 扩写
- `~~删除线~~` → 简写
- `*斜体*` → 润色（注意：单个 `*` 是斜体，`**` 是加粗，区分优先级）

如果正文中没有任何标记，提示用户：

```text
未检测到返修标记。请在正文中加入标记后重新触发返修：
- **加粗** = 扩写（一句话太干）
- ~~删除线~~ = 简写（太啰嗦）
- *斜体* = 润色（不顺/散/卡）
```

## 第三步：逐个处理标记

### 3.1 SSoT 候选生成

对每个标记，在心里生成一个随机字符串（至少 20 字符），分 3 段分别驱动 3 个候选版本：

```
随机字符串: "kR7pL2mN4bT9xQwJ5aHc"
├── 前 7 字符 "kR7pL2m" → 候选 A 的策略选择和微决策
├── 中 7 字符 "N4bT9xQ" → 候选 B 的策略选择和微决策
└── 后 6 字符 "J5aHc"   → 候选 C 的策略选择和微决策
```

每段通过 Sum-Mod 运算驱动：

1. **策略选择**：从该标记类型的策略池中选一种（见 `{SKILL_DIR}/references/revision-rules.md`）
2. **句式种子**：决定首句结构（动作开头/感官开头/对话开头/陈述/省略）
3. **感官种子**：偏重哪类感官细节（视觉/听觉/触觉/嗅觉/本体感觉）
4. **词汇种子**：从同义候选池中选哪个词

**核心约束**：3 个候选必须分别走不同策略，不允许两个候选走同一种策略。

计算过程在 thinking 中完成，不暴露给用户。

### 3.2 策略池概览

每种标记类型有 3 种策略，确保 3 个候选分别走不同路径。完整策略定义和进阶操作见 `{SKILL_DIR}/references/revision-rules.md`。

**扩写（加粗）策略**：

| 候选 | 策略 | 核心手法 |
|------|------|---------|
| A | 动作展开 | 拆复合动作为序列，加操作步骤和具体化描写 |
| B | 人物反应层 | 加旁观者反应，插入对话交锋，增加现场张力 |
| C | 感官与内心 | 加感官铺陈，加口语化内心独白，增加沉浸感 |

**简写（删除线）策略**：

| 候选 | 策略 | 核心手法 |
|------|------|---------|
| A | 合并压缩 | 多短句合并为一个通顺长句，保留所有信息点 |
| B | 概括替代 | 用一个精准的动作/判断替代多句铺陈 |
| C | 留白裁剪 | 删掉修饰和过程，只留关键动作和结果 |

**润色（斜体）策略**：

| 候选 | 策略 | 核心手法 |
|------|------|---------|
| A | 补过渡 | 加感知步骤、因果连接词、自然过渡，让句子顺滑 |
| B | 句式重构 | 调整语序、改变句式结构、加入结构词 |
| C | 感官化改写 | 用身体感受和具体细节替代抽象或干硬的描述 |

### 3.3 用户交互

直接以文字输出候选版本，不使用 `AskUserQuestion` 工具：

```text
【返修标记 #{N}】{位置描述}
原文：{被标记的原文}
操作：{扩写/简写/润色}

A. {候选版本 A}
B. {候选版本 B}
C. {候选版本 C}

请选择 A/B/C，或直接输入自定义改写。
```

等待用户回复后，继续处理下一个标记。

### 3.4 写入规则

用户选择后：

1. 用选定版本替换正文中被标记的文字
2. 移除标记格式（加粗/删除线/斜体的 markdown 语法）
3. 立即写回 `正文.md`
4. **重新读取文件**后再处理下一个标记（因为文件内容已变更）

### 3.5 循环返修

所有标记处理完毕后，用户可以再次在正文中添加标记并重新触发返修。

## 第四步：交付与提示

所有标记处理完毕后输出：

```text
✅ 返修完成

处理了 {N} 个标记：
- 扩写：{N} 处
- 简写：{N} 处
- 润色：{N} 处

📝 下一步：
- 如需继续返修，在正文中添加新标记后再次运行 /novel-revise
- 如确认满意，运行 /novel-sync chapter {N} 同步知识图谱
```

## 关键原则

1. **只动标记处** — 未标记的文字一个字都不改
2. **人做选择** — AI 生成所有候选，人来挑
3. **SSoT 保多样** — 同一章节同类型标记的候选不会趋同
4. **立即写入** — 选定后立即写回文件，不积压
5. **不改结构** — 如果发现需要改动剧情结构，回到 `novel-plan --revise`
6. **不做 AI 味复查** — 返修后直接交付，不再跑 AI 味检测

## 与其他 Skill 的关系

```
novel-write（完成正文 + AI 味检测 + 润色 + 仿人工增强）
      ↓
novel-revise（返修）← 你在这里
      ↓
novel-sync（同步状态）
      ↓
[继续创作下一章]
```
