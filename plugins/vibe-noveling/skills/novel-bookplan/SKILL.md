# 全书与分卷剧情规划

## 职责边界

负责全书级与分卷级节奏，不负责单章落纲。核心流程：**诊断故事母型 → 建立全书节奏蓝图 → 拆成分卷节奏卡 → 写入 memory/future/**。

方法论参考 Save the Cat（15 Beats + 10 种故事母型），详见 `references/stc-story-types.md` 和 `references/stc-hierarchy.md`。

分层原则：
- **全书层**：完整主母型、主 beat 分布、终局承诺
- **分卷层**：每卷承接全书一段 beat 或形成"卷内迷你弧"
- 全书规划阶段不预设章节数，优先用卷、职责段/卷内位置/状态变化描述结构

## 初始化

读取 `CLAUDE.md` 创作进度和风格基线，以及 `memory/future/` 下已有文件。

首次规划（`00-index.md` 不存在）时，先初始化 future/：问用户终极目标（主角最终变成什么样？解决什么核心矛盾？获得或失去什么？），创建最小化 `10-book.md`。

已有规划时先展示当前状态（故事母型、全书节奏蓝图有无、分卷节奏卡数量、伏笔数、修订版本）。

## 主菜单

```
A. 从零规划全书 — 故事母型 + 全书 beats + 卷结构
B. 诊断故事母型 — 只做母型识别与选择
C. 规划某一卷 — 在已有全书蓝图下细化单卷
D. 补完设定后重排 — 根据新设定局部或整体重校节奏
E. 查看节奏蓝图 — 查看全书 beats、分卷职责、伏笔时间线
F. 管理 future/ — 记录新想法、整理伏笔、维护修订记录
```

---

## A. 从零规划全书

### A1. 全书定位
确认类型/基调/目标读者/预计体量。长篇默认全书 STC + 分卷节奏卡。

### A2. 故事母型诊断
读 `references/stc-story-types.md`，基于设定给最匹配的 3 个母型候选（含为什么像、不像另外几个、最吃香的卷、可能踩的坑）。主母型只能有一个，副母型可选。

### A3. 一句话 → 一段话展开
基于主母型生成一句话概括（含主角身份、核心困境、读者承诺、母型气质），确认后扩展为 5 句话对应 5 个 Act。

### A4. 全书节奏蓝图
建立完整 15 beats 分布。长篇额外标注对应卷、卷内位置/职责段、核心事件、关键状态变化、是否锁定。不写"大致章节范围"。

### A5. 分卷节奏卡
把全书 beats 拆成分卷职责。每卷含：卷目标、卷功能、承接全书 beat、起点状态、Catalyst、Midpoint、坠点/最低谷、卷末钩子、回收/埋设伏笔。卷不一定有完整 15 beats，但必须有明确起点、升级点、卷末变化、卷末钩子。

### A6. 角色弧线 + 伏笔时间线
为主要角色补弧线（动机/目标/冲突/顿悟），检查顿悟点是否与全书关键 beat 对齐。输出伏笔埋设/揭示表和困境层次递进表。

### A7. 黄金三章（如需要）
设计前三章的钩子、承诺感和追更理由。非三章型开篇就直说，不机械压成三章。

### A8. 审阅与保存
通过 4 项校验（主母型贯穿、beats 有落点、分卷职责不重复、允许后续局部修订）后写入 `10-book.md`、`20-threads.md` 与对应卷卡。

---

## B. 诊断故事母型

已有粗大纲但不知道像哪种故事时使用。读 `references/stc-story-types.md`，基于当前设定给 3 个最匹配母型，说明承诺感和风险。选定后写入 `10-book.md`。

---

## C. 规划某一卷

### C1. 先读全书蓝图
不允许脱离全书单独造卷。输出当前卷定位（承接全书 beat、前一卷结束状态、本卷应回答/不能提前揭开的内容）。

### C2. 卷目标与卷承诺
明确主角在这卷完成什么、读者想看什么承诺感、主要冲突类型、卷末必须发生的变化。

### C3. 卷内节奏卡
短卷/过桥卷用压缩卡（起点-升级点-反转-坠点/胜利-钩子），主力大卷用完整卷内 STC。

### C4. 拆职责段
列卷内职责段（卷首新局面 → 中前段兑现承诺感 → 关键转折段反转 → 中后段压力加码 → 卷末坠点/胜利+新钩子），不预设每段落几章。保存到 `30-volumes/vol-xx.md`。

---

## D. 补完设定后重排

### D1. 列出新增设定 + 影响评估
按 4 类判定：不影响节奏只影响细节 / 影响某卷冲突设计 / 影响某几个 beat 落点 / 影响主母型承诺需警惕漂移。

### D2. 给 3 种重排策略
A 最小改动 / B 中等重排 / C 结构升级。区分锁定项（已写入正文或已确认的终局承诺）和可改项（未写到的卷结构、伏笔时机、出场顺序）。禁止未经提醒推翻已写成正文的关键结构。

### D3. 写修订记录
触发原因、保留内容、调整内容、影响卷。

---

## E. 查看节奏蓝图

表格展示全书 beats（Beat / 对应卷 / 卷内位置 / 关键事件 / 状态），分卷节奏卡索引，伏笔时间线，修订记录。

---

## F. 管理 future/

支持记录远期想法、整理过期条目、合并重复伏笔、更新修订记录、标记确认/草案状态。按卷/beats/职责段维护，不把章节数量当整理前提。

---

## 保存

```
memory/future/
├── 00-index.md          ← 当前卷/事件/结构说明
├── 10-book.md           ← 终极目标、母型、一句话/一段话、节奏蓝图、困境升级、修订记录
├── 20-threads.md        ← 长期线程总表
├── 30-volumes/vol-xx.md ← 单卷目标/功能/节奏卡/线程安排
├── 40-events/           ← 事件中程规划
└── 90-sync-tracker.md   ← novel-sync 维护的伏笔追踪
```

权限：novel-bookplan 写 future/；novel-plan 只读 future/；novel-sync 只更新 threads 状态和 sync-tracker，不重写结构。

## 禁止

- 每章硬套完整 15 beats
- 不读全书蓝图就单独规划某卷
- 补完设定后沿用旧节奏不做影响评估
- 未经用户确认推翻已锁定终局或已写入正文的重大结构
- 把 Save the Cat 当死板模板，忽略故事母型匹配
