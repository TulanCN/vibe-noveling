# 全书与分卷剧情规划

## 目标

通过苏格拉底式对话，帮助用户做三类规划：

- **全书层**：故事母型、15 beats、卷结构、终局方向
- **分卷层**：单卷目标、卷内节拍、卷末钩子、伏笔回收与埋设
- **重规划层**：补完设定后，定位受影响的节拍与卷，做局部或整体重排

> **核心流程**：诊断故事母型 → 建立全书节奏蓝图 → 拆成分卷节奏卡 → 写入 `memory/future/` → 后续由 `novel-plan` 读取执行

## 使用方式

```bash
/novel-bookplan
/novel-bookplan --view
/novel-bookplan --future
/novel-bookplan --volume Vol-02
/novel-bookplan --replan
```

## 方法论参考

使用 Save the Cat 相关规划前，按需读取以下 reference：

- `references/stc-story-types.md`：10 种故事母型的定义、识别问题、网文适配方式
- `references/stc-hierarchy.md`：全书层 / 分卷层 / 章节职责层的分层用法

### 雪花写作法核心理念

从一句话开始，逐层扩展：1 句话 → 1 段话 → 角色概要 → 卷梗概 → 场景职责。
每一步都允许回溯修改，追求结构清晰和可迭代，而不是一次到位。

### Save the Cat 15 Beats

| # | Beat | 位置 | 定义 |
|---|------|------|------|
| 1 | Opening Image | 0-1% | 主角"之前"状态 |
| 2 | Theme Stated | 5% | 暗示主题的话语 |
| 3 | Setup | 1-10% | 日常世界、缺陷、初始目标 |
| 4 | Catalyst | 10% | 改变人生的事件 |
| 5 | Debate | 10-20% | 犹豫和反应 |
| 6 | Break Into Two | 20% | 踏入新世界 |
| 7 | B Story | 22% | 承载主题的副线/人物 |
| 8 | Fun and Games | 20-50% | 读者为这个题材买单的承诺感 |
| 9 | Midpoint | 50% | 虚假胜利/虚假失败 |
| 10 | Bad Guys Close In | 50-75% | 内外压力收紧 |
| 11 | All Is Lost | 75% | 最低谷 |
| 12 | Dark Night of the Soul | 75-80% | 灵魂暗夜 |
| 13 | Break Into Three | 80% | 顿悟时刻 |
| 14 | Finale | 80-99% | 蜕变证明 |
| 15 | Final Image | 99-100% | "之后"画面 |

### 分层 STC 原则

- **全书层**：必须有完整的主母型、主 beat 分布、终局承诺
- **分卷层**：每卷要么承接全书的一段 beat，要么形成一段"卷内迷你弧"
- **章节层**：不要强行每章都塞满 15 beat；只定义本章承担的节奏职责
- **规划边界**：全书规划阶段不预设章节数，优先使用卷、职责段 / 卷内位置 / 状态变化来描述结构

当用户只想规划某一卷时，也要先读取全书层的主母型和 beat 分布，避免单卷失焦。

## 执行流程

### 第零步：初始化检查

每次进入规划前，先读取并汇总：

- `memory/future/00-index.md`
- `memory/future/10-book.md`
- `memory/future/20-threads.md`
- `memory/future/30-volumes/`
- `memory/future/40-arcs/`
- `memory/setting-todo.md`（如存在）
- `memory/worldbuilding.md`（如存在）
- `memory/discussions.md`（如存在）
- `memory/entities/characters/`
- `memory/entities/factions/`
- `memory/entities/concepts/`
- `memory/entities/locations/`
- `memory/entities/items/`

检查重点：

1. `memory/future/00-index.md` 和 `memory/future/10-book.md` 是否存在
2. 是否已有「全书节奏蓝图」或旧版「全书大纲」
3. 是否已有「分卷节奏卡」
4. 最近是否补完了会影响剧情骨架的新设定

#### A. future/ 初始化

如果 `memory/future/00-index.md` 不存在，先初始化 `memory/future/`：

```text
🆕 首次规划 - 初始化 future/

检测到尚无 future/ 规划目录，需要先确定故事终局方向。

【终极目标】主角最终会变成什么样？

可以从三个维度回答：
1. 最终达到什么境界/地位？
2. 最终解决什么核心矛盾？
3. 最终获得什么，或失去什么？
```

根据用户回答创建最小化的 `memory/future/10-book.md`，并保留后续扩展空间。

#### B. 规划状态展示

如果 `memory/future/00-index.md` 已存在，先输出：

```text
📖 当前规划状态

future/：已存在
   故事母型：主母型 [X] / 副母型 [Y 或空]
   全书节奏蓝图：✅/❌
   分卷节奏卡：X 卷
   伏笔：X 条
   修订版本：vX
   最近补充设定：[摘要]
```

如果只有旧版全书大纲，没有故事母型和分卷节奏卡，要明确提示：

```text
⚠️ 检测到旧版全书规划

已有：一句话概括 / 卷结构 / 伏笔
缺少：故事母型 / 分卷节奏卡 / 修订记录

建议先做一次结构升级，再继续规划后续卷。
```

## A 流程：从零规划全书

### A1. 全书定位

先确认：

- 类型 / 子类型
- 基调
- 目标读者
- 预计字数或卷数
- 这是偏单核主线，还是多卷升级型长篇

不要主动要求用户提供总章节数。全书层只需要体量感，不需要先把所有事件绑定到具体章位。

如果是长篇连载，默认采用：

- 全书层：完整 STC
- 分卷层：每卷设节奏卡
- 章节层：由 `novel-plan` 只读职责，不重复造骨架

### A2. 故事母型诊断

读取 `references/stc-story-types.md`，基于现有设定给出最匹配的 3 个母型候选，而不是让用户从 10 个里盲选。

输出格式：

```text
🧭 故事母型诊断

最匹配的 3 个候选：

方案 A：【母型名】
   核心承诺：
   为什么像：
   风险：

方案 B：【母型名】
...

方案 C：【母型名】
...
```

要求用户选择：

- **主母型**：决定全书主承诺
- **副母型**：可选，用于增强某些卷的气质

如果用户坚持混合多种母型，要提醒：

```text
⚠️ 母型可以混搭，但主母型只能有一个。
否则全书承诺会漂。
```

### A3. 一句话概括

基于主母型，生成 2 个更贴承诺感的一句话概括，让用户二选一或修改。

输出时要显式检查一句话是否体现：

- 主角身份
- 核心困境
- 读者承诺
- 母型气质

### A4. 一段话展开

将一句话扩展为 5 句话，分别对应：

- 句 1：Act 1 / Setup + Catalyst
- 句 2：Act 2A / 新世界 + 承诺感
- 句 3：Midpoint
- 句 4：Act 2B / 压力升级 + 低谷
- 句 5：Act 3 / 终局与结果

这里要同时检查：这 5 句话是否仍符合主母型承诺。

### A5. 全书节奏蓝图

建立完整的全书 beat 分布：

```text
📊 全书节奏蓝图

Opening Image：
Theme Stated：
Setup：
Catalyst：
Debate：
Break Into Two：
B Story：
Fun and Games：
Midpoint：
Bad Guys Close In：
All Is Lost：
Dark Night：
Break Into Three：
Finale：
Final Image：
```

如果是长篇多卷小说，要额外标注：

- 对应卷
- 对应的卷内位置 / 职责段
- 该 beat 的核心事件
- 关键状态变化
- 是否已锁定，还是允许后续重排

不要在全书 beat 蓝图里写“大致章节范围”或“第几章左右”。这类章位信息留给 `novel-plan` 落单章时再决定。

### A6. 分卷节奏卡

读取 `references/stc-hierarchy.md`，把全书 beats 拆成分卷职责。

每卷都要形成一张节奏卡：

```text
📖 Vol-01 节奏卡

卷目标：
卷功能：它在全书里承担什么职责
承接的全书 beat：
卷内起点状态：
卷内 Catalyst：
卷内 Midpoint：
卷内坠点/卷末最低谷：
卷末钩子：
本卷回收哪些伏笔：
本卷埋设哪些伏笔：
```

说明原则：

- 卷不一定必须独立拥有完整 15 beats
- 但每卷必须有明确的起点、升级点、卷末变化、卷末钩子
- 如果某卷篇幅很长，可以为该卷建立"完整卷内 STC"

### A7. 角色弧线与节奏校验

为主要角色补上：

- 一句话弧线
- 动机 / 目标 / 冲突 / 顿悟
- 在全书 STC 中承担的功能
- 在哪些卷会推进或反转

检查角色顿悟点是否与主角的 Break Into Three、Finale 等关键 beat 对齐。

### A8. 伏笔时间线 + 困境层次

输出：

- 主线伏笔埋设/揭示表
- 卷级伏笔回收表
- 困境层次递进表

要求每卷至少明确：

- 本卷新增什么疑问
- 本卷回答什么旧问题
- 本卷把哪条线继续推迟

### A9. 开篇策略 / 黄金三章

如果用户在做首次全书规划，或明确提到"黄金三章"，要额外设计：

- 第一章必须抛出的主钩子
- 第二章必须扩大的承诺感
- 第三章必须建立的稳定追更理由

输出格式：

```text
🏆 黄金三章策略

Ch1：
- 必须完成：
- 章末钩子：

Ch2：
- 必须完成：
- 章末钩子：

Ch3：
- 必须完成：
- 章末钩子：
```

如果故事不是前三章型开篇，而是"开篇卷前 10 章渐入佳境"，也要明确说明，不要机械压成三章。

### A10. 审阅与保存

统一做 4 项校验：

```text
🔍 结构校验

□ 主母型是否清晰，且贯穿全书承诺
□ 15 beats 是否都有落点
□ 分卷是否职责明确，而不是重复同一种卷
□ 后续是否允许在不推翻全书的前提下继续局部修订
```

通过后写入 `memory/future/10-book.md`、`memory/future/20-threads.md` 与对应卷卡。

## B 流程：诊断故事母型

适合场景：

- 用户已有粗大纲，但不知道像哪一种故事
- 用户觉得现有节奏单一，想重新找主承诺
- 用户打算先定母型，再决定卷结构

执行方式：

1. 读取 `references/stc-story-types.md`
2. 基于当前设定给出 3 个最匹配母型
3. 说明每个母型对长篇网文的"承诺感"
4. 让用户选择主母型和可选副母型
5. 将结果写入 `memory/future/10-book.md`，即使暂时不做全书大纲也要记录

输出必须包含：

- 为什么像
- 为什么不像另外几个
- 如果选这个母型，哪些卷会最吃香
- 可能踩的坑

## C 流程：规划某一卷

适合场景：

- 全书已经有大概方向，想单独细化某一卷
- 后续补完设定后，准备回来细化 Vol-02、Vol-03
- 某卷需要从"只有一句话概括"升级成可执行的卷蓝图

### C1. 先读全书蓝图，不允许脱离全书单独造卷

输出：

```text
📍 当前卷定位

所属卷：Vol-02
主母型：
该卷承接的全书 beat：
前一卷结束状态：
本卷应回答的问题：
本卷不能提前揭开的内容：
```

如果全书蓝图里没有这卷的职责，明确提示用户先补全书层，或由本流程顺手补上。

### C2. 设计卷目标与卷承诺

本卷至少要明确：

- 主角在这卷要完成什么
- 读者在这卷想看什么承诺感
- 这卷的主要冲突类型
- 这卷结束时必须发生什么变化

### C3. 建立卷内节奏卡

根据该卷篇幅，二选一：

- **短卷/过桥卷**：使用压缩节奏卡
- **主力大卷**：使用完整卷内 STC

压缩节奏卡最少包含：

- 卷起点
- 卷内升级点
- 卷中反转
- 卷末坠点或卷末胜利
- 卷末钩子

### C4. 拆成本卷职责段

不要直接写具体章节剧情，而是先列职责段：

```text
本卷职责段建议

第 1 段（卷首）：建立新局面 / 新规则
第 2 段（卷中前段）：兑现承诺感
第 3 段（卷中关键转折段）：反转
第 4 段（卷中后段）：压力加码
第 5 段（卷末）：坠点/胜利 + 新钩子
```

这一步是给 `novel-plan` 用的，不是替代 `novel-plan`。
`novel-bookplan` 只需要给出职责段和结构锚点，不需要先决定这一段会落成几章。

### C5. 保存卷卡

保存到 `memory/future/30-volumes/vol-xx.md`，并同步更新 `memory/future/10-book.md` 的修订记录。

## D 流程：补完设定后重排

这是长篇项目的常用流程，必须作为正式场景支持。

适合场景：

- 第一次只做了粗略全书规划
- 后续补完世界观 / 势力 / 修炼体系 / 角色动机
- 发现原来的卷结构、伏笔时机、角色弧线已经不够贴

### D1. 先列出新增设定

输出：

```text
🧩 设定变更摘要

新增/修改：
1. [设定]
2. [设定]
3. [设定]
```

如果用户没有显式总结，主动从 `memory/worldbuilding.md`、实体文件和 `memory/discussions.md` 提炼。

### D2. 做影响评估

按 4 类影响输出：

- 不影响节奏，只影响表现细节
- 影响某一卷的冲突设计
- 影响某几个 beats 的落点
- 影响主母型承诺，需要警惕整体漂移

### D3. 给出 3 种重排策略

```text
方案 A：【最小改动】
只调 Vol-02、Vol-03 的节奏卡，不动主母型和终局

方案 B：【中等重排】
保留主母型和终局，重排中段 beats 与伏笔时机

方案 C：【结构升级】
重新审视主母型、副母型和卷职责，保留核心设定，重做节奏分布
```

先让用户选重排力度，再动结构。

### D4. 标记锁定项与可改项

明确区分：

- **锁定项**：已经写入正文或已经确认不改的终局承诺
- **可改项**：尚未写到正文的卷结构、伏笔揭示时机、角色出场顺序

禁止在未提醒用户的情况下，推翻已写成正文的关键结构。

### D5. 写修订记录

每次重排都要写入：

```markdown
## 规划修订记录

### v2
- 触发原因：
- 保留内容：
- 调整内容：
- 影响卷：
```

## E 流程：查看节奏蓝图

输出优先使用表格，方便用户快速扫读：

```text
📚 全书节奏蓝图

主母型：
副母型：

| Beat | 对应卷 | 卷内位置/职责段 | 关键事件 | 状态 |
|------|--------|-----------------|---------|------|
```

然后展示：

- 分卷节奏卡索引
- 伏笔时间线
- 修订记录摘要

如果发现某卷没有节奏卡，直接提示补齐。

## F 流程：管理 future/

支持：

- 记录新的远期想法
- 整理已过期条目
- 合并重复伏笔
- 更新修订记录
- 标记哪些卷节奏卡已确认，哪些仍是草案
- 按卷、beats、职责段维护 future/，不把章节数量当作整理前提

## 保存格式

将结果保存到 `memory/future/` 对应文件，建议采用以下分工：

```text
memory/future/
├── 00-index.md
│   - 当前卷 / 当前 arc / 结构说明 / 迁移说明
├── 10-book.md
│   - 终极目标
│   - Save the Cat 故事母型
│   - 一句话概括 / 一段话展开
│   - 全书定位
│   - 全书节奏蓝图
│   - 困境升级
│   - 修订记录
├── 20-threads.md
│   - 主线线程总表（长期伏笔、主线矛盾、终局线）
├── 30-volumes/
│   ├── vol-01.md
│   ├── vol-02.md
│   └── ...
│   - 单卷目标 / 卷功能 / 结构蓝图 / 本卷线程安排
├── 40-arcs/
│   - 当前或重点 arc 的中程规划
└── 90-sync-tracker.md
    - 仅供 novel-sync 维护的章节伏笔追踪
```

## 与其他 Skill 的关系

```text
novel-bookplan（故事母型 + 全书/分卷节奏）
     │ 输出到 memory/future/
     ↓
novel-plan（逐章规划）
     │ 读取：10-book / 20-threads / 30-volumes / 40-arcs
     ↓
novel-write（章节创作）
     ↓
novel-sync（同步 past.md / future/ 状态）
```

**权限划分**：

- `novel-bookplan`：`memory/future/` 写权限
- `novel-plan`：`memory/future/` 只读，用于读取节奏职责
- `novel-sync`：仅更新 `20-threads.md` 状态和 `90-sync-tracker.md`，不重写结构

## 关键原则

1. **先定故事母型，再定节奏骨架**，不要反过来
2. **全书层和分卷层分开设计**，不要把整本书压成一张卷表
3. **章节层只定义职责，不强塞完整 15 beats**
4. **长篇要允许重规划**，第一次全书规划不是终稿
5. **重排时先做影响评估**，再决定改动力度
6. **每卷都要有存在意义**，不能只是重复上一卷的换皮升级
7. **伏笔必须配揭示时机**，不能只堆未兑现的悬念
8. **结构服务承诺感**，不是为了把表格填满
9. **用户确认后再保存**，尤其是主母型、终局和重排策略

**禁止**：

- 把 Save the Cat 只当成唯一死板模板，不看故事母型
- 每章都硬套完整 15 beats
- 不读全书蓝图就单独规划某卷
- 补完设定后还沿用旧节奏却不做影响评估
- 未经用户确认就推翻已锁定的终局或已写入正文的重大结构
