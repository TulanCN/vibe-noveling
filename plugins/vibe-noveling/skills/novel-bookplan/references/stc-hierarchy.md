# Save the Cat 分层规划指南

用途：

- 把 Save the Cat 从"只用于全书"升级成"全书层 + 分卷层 + 章节职责层"
- 解决长篇网文不能把每一章都硬套完整 15 beats 的问题

## 总原则

Save the Cat 在长篇里的正确用法不是：

- 每章一个完整 15 beats

而是：

- **全书层**：完整 15 beats，决定主结构
- **分卷层**：承接全书某一段 beats，或形成卷内迷你弧
- **章节职责层**：只定义本章承担什么节奏职责

## 一、全书层

全书层必须明确：

- 主母型 / 副母型
- 全书 15 beats 的落点
- 各卷分别承担哪些 beats
- 终局兑现什么核心承诺

适合回答的问题：

- 这本书到底是什么故事
- 为什么读者会持续追更
- 每卷在大结构里的功能是什么

### 长篇常用映射

| 全书区段 | 常见卷位 | 说明 |
|---------|----------|------|
| Opening Image ~ Debate | Vol-01 | 建世界、立困境、抛 Catalyst |
| Break Into Two ~ Fun and Games | Vol-02 | 进入新局面，兑现题材承诺 |
| Midpoint ~ Bad Guys Close In | Vol-03 | 局面升级，代价变重 |
| All Is Lost ~ Finale | Vol-04+ | 低谷、顿悟、总决战、结算 |

这只是默认映射。不要机械套用。卷数多时，要把一个大区段拆成多卷子弧。

## 二、分卷层

分卷层的任务不是复制全书，而是把全书节奏拆成可执行的阶段。

每卷至少回答 6 个问题：

1. 这卷在全书里承担什么功能
2. 这卷要兑现什么承诺感
3. 这卷的起点状态是什么
4. 这卷的升级点或中点反转是什么
5. 这卷结尾要把主角推到什么新状态
6. 这卷结尾留什么钩子给下一卷

### 什么时候给一卷完整 STC

满足以下任一情况，可以为该卷建立完整卷内 15 beats：

- 该卷篇幅很长
- 该卷本身像一部独立小小说
- 该卷承担大幅度角色关系变化
- 该卷是全书中的核心大战役 / 核心旅程 / 核心真相卷

### 什么时候只用压缩节奏卡

适合以下情况：

- 过桥卷
- 调整世界格局的过渡卷
- 篇幅较短的承上启下卷
- 重点只是把角色从 A 状态送到 B 状态

压缩节奏卡建议包含：

- 卷起点
- 卷内升级点
- 卷中反转
- 卷末坠点或卷末胜利
- 卷末钩子

## 三、章节职责层

章节层不要再硬套完整 STC。章节的正确问题是：

- 这章在本卷里负责什么
- 这章推进哪个 beat 或哪段承诺感
- 这章结束时，状态发生什么微变化

### 可用的章节职责标签

| 标签 | 用途 |
|------|------|
| Opening / Setup 章 | 建立当前局面、目标、规则 |
| Catalyst 章 | 让事情真正发生变化 |
| Debate 章 | 让主角犹豫、试探、被迫选择 |
| Break 章 | 主角踏入新局面 |
| Fun and Games 章 | 兑现题材承诺、给读者爽点或新鲜感 |
| B Story 章 | 推进关系线、主题线、搭档线 |
| Midpoint 预热章 | 为中点反转做铺垫 |
| Midpoint 章 | 改变局面认知 |
| Pressure 章 | 加码、压缩退路 |
| All Is Lost 章 | 把主角推到底 |
| Dark Night 章 | 低谷后的沉降、重组 |
| Break Into Three 章 | 顿悟、找到新方案 |
| Finale 章 | 执行新方案并证明蜕变 |
| Hook 章 | 章末留下驱动下一章的钩子 |

一个章节可以有 1 个主职责和 1 个副职责，但不要超过 2 个。

## 四、章节数很多时怎么切

长篇网文章节很多，但 `novel-bookplan` 不需要先把它们定成具体数量。常见做法是先按"职责段"切，再进入逐章规划。

示例：

```text
Vol-02 职责段

第 1 段（卷首职责段）：Break Into Two 后的新局面建立
第 2 段（卷中前段）：Fun and Games，兑现题材承诺
第 3 段（卷中关系推进段）：B Story 加深，关系推动
第 4 段（卷中关键转折段）：Midpoint 预热和爆发
第 5 段（卷中后段）：Bad Guys Close In
第 6 段（卷末职责段）：卷末反转 + 钩子
```

`novel-plan` 再去把其中某一职责段拆成具体章节。也就是说，全书和分卷规划阶段先回答“这段负责什么”，单章阶段再回答“这一章是多少章”。

## 五、补完设定后怎么重排

当世界观、势力、规则体系、角色动机被补完后，优先检查：

1. 主母型是否仍然成立
2. 哪些卷的承诺感被改变
3. 哪些 beats 的落点不再自然
4. 哪些章节职责需要挪位

### 建议重排顺序

1. 先改全书层：确认主母型、关键 beats、终局不变还是变
2. 再改分卷层：哪些卷功能要调整
3. 最后改章节职责层：哪些章要补、删、合并、前移、后移

不要反过来从单章补丁开始修，否则会越来越乱。

## 六、常见错误

### 错误 1：把每一卷都写成同一套升级模板

表现：

- 每卷都是到新地方、打新人、升一级、留钩子

修正：

- 明确每卷承担的结构功能不同
- 有的卷负责承诺感，有的卷负责真相，有的卷负责关系裂变

### 错误 2：每章都想承担太多职责

表现：

- 一章里既想世界观展示、又爽、又反转、又推进感情、又埋伏笔

修正：

- 每章只保留 1 个主职责和 1 个副职责

### 错误 3：把分卷层缺口推给逐章规划

表现：

- 卷没有目标，直接让 `novel-plan` 规划下一章

修正：

- 先补卷目标和卷节奏卡，再做逐章规划

## 七、推荐工作流

1. `novel-bookplan` 选主母型 / 副母型
2. `novel-bookplan` 建全书节奏蓝图
3. `novel-bookplan` 拆分卷节奏卡
4. `novel-bookplan` 给出某卷的职责段
5. `novel-plan` 按职责段规划下一章
6. `novel-sync` 随写作进度更新状态
