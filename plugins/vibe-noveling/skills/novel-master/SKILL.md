# 小说风格控制器

## 核心理念

**两重防御：AI 味检测 + 反常规检查**

- 第一层：AI 味检测 — 消除所有"一看就是 AI 写的"痕迹
- 第二层：反常规检查 — 确保剧情不会平淡无聊

## 一、AI 味检测清单（首要检查）

写作完成后，逐条检查以下 10 种 AI 写作痕迹。

### 检测项 1：过度心理描写

```
🚩 AI 味：
   小明心中充满了矛盾和纠结，他一方面觉得自己应该勇敢面对，
   另一方面又害怕失败后的嘲笑，这种复杂的情绪让他一时间难以做出决定。
   他回想起从小到大每一次被质疑的经历，那些画面如走马灯般在脑海中闪过……

✅ 修正：只留最关键的一句
   小明的手心全是汗。
   他攥了攥拳头，又松开。
   "没事。"他说。声音比自己预想的要稳。
```

### 检测项 2：所有人都很客气

```
🚩 AI 味：
   "苏师弟，你这样做不太好吧？我认为我们应该好好商量一下。"
   "你说得有道理，我也觉得可以再讨论讨论。"

✅ 修正：给对话加"刺"
   "你配？"
   只有两个字。小华甚至没有回头。
```

### 检测项 3：万能形容词泛滥

```
🚩 AI 味（出现以下词语要警觉）：
   "深邃的目光""沉稳的声音""坚定的步伐""复杂的表情"
   "强大的气场""莫名的危机感""说不出的感觉"

✅ 修正：用具体动作代替形容词
   ❌ 老李的眼神很深邃
   ✅ 老李看着他，看了很久，没说话
```

### 检测项 4：总结式结尾

```
🚩 AI 味：
   小明望着远方，他知道，这只是开始。
   月光洒在少年身上，映照着他坚毅的脸庞。
   他暗暗发誓，一定要变强。

✅ 修正：用悬念或动作结尾
   小华走了。
   小明没有发誓，也没有咬牙。
   他只是站在那里，很久。
   然后转身，继续练剑。
```

### 检测项 5：均衡分配，没有重点

```
🚩 AI 味：
   每个场景占差不多字数，每个角色都有差不多戏份，
   战斗、对话、描写、心理各占 25%，像配餐一样均衡。

✅ 修正：放大重点，压缩过渡
   关键战斗写 1500 字
   过渡对话写 200 字
   不要平均分配
```

### 检测项 6：情感标签替代行为描写

```
🚩 AI 味：
   他感到愤怒。他感到悲伤。他心中涌起一股喜悦。
   她的脸上露出了复杂的表情。

✅ 修正：用行为展示情感
   ❌ 他感到愤怒
   ✅ 他一拳砸在桌上。茶杯震了三下。

   ❌ 她露出了复杂的表情
   ✅ 她张了张嘴，又闭上了。最后什么都没说。
```

### 检测项 7：解释性对话

```
🚩 AI 味：
   "你知道为什么筑基期很难突破吗？因为需要将灵气从气态
    转化为液态，这个过程需要……"
   "原来如此，我明白了！"

✅ 修正：用冲突和行动传递信息
   "凝气九重？"
   长老的手顿了一下。
   "你确定？"
```

### 检测项 8：不必要的过渡段

```
🚩 AI 味：
   三天后，小明来到了灵药园。
   时间过得很快，转眼间已经是秋天。
   经过一番准备，他终于出发了。

✅ 修正：直接切入下一个场景
   灵药园。
   小明蹲在一株灵草前，已经等了三个时辰。
```

### 检测项 9：修辞堆砌

```
🚩 AI 味（一段话里超过一个比喻/拟人）：
   灵力如同汹涌的洪流在他体内奔涌，仿佛一头被释放的猛兽，
   又像是要将他的经脉撕裂的烈焰，他的身体像是被放在了砧板上的鱼肉。

✅ 修正：一段最多一个修辞
   灵力在经脉中奔涌。
   痛。
   像有千万根针同时扎入骨髓。
```

### 检测项 10：所有人都很理性

```
🚩 AI 味：
   每个角色都能理性分析局势，每个决定都有合理动机，
   反派也是讲道理的，冲突都是"误会"。

✅ 修正：让人更任性、更极端、更不讲理
   "把东西交出来。"
   "这是我的。"
   "我再问一遍。"
   "不。"
   拳头砸了下来。
```

## 一·五、高频警示词列表

以下词汇/句式在写作中出现频率过高，容易产生"AI 味"或"翻译腔"。**不是绝对禁用，而是优先寻找替代表达**。

### 精确警示词（出现时必须考虑替换）

- 过渡词（值得注意的是、综上所述、本质上、不容置疑）→ 删掉，或用动作衔接
- 动作描写（摩挲、闪过、微微、轻微）→ 用具体动作（"手指在桌上敲了两下"替代"微微紧张"）
- 神态描写（挑眉、勾起嘴角、深吸一口气）→ 删掉或用更具体的（"嘴角动了一下"替代"勾起嘴角"）
- 声音描写（金属摩擦般的质感、低声重复）→ 用具体的声音（"刺啦"替代"金属摩擦般的质感"）
- 形容词（坚毅、坚定、警惕、惊恐）→ 用动作展示（"握紧了剑"替代"警惕地看着"）
- 比喻词（好像、似的、如同、仿佛，单段超2个）→ 换用直述或换比喻载体
- 填充词（一丝、有些、不易察觉）→ 删掉（"一丝不安"→不安直接写行为）
- 面部（面庞、额角、发光）→ 换用更具体的部位或删掉
- 环境陈词（融入夜色中、夜色如墨）→ 用具体的环境细节替代
- 身体细节（茧子、老茧、骨节、肩胛骨、胯骨）→ 删掉，或用具体动作替代
- 衣物细节（衣服下摆、布料、衣料）→ 删掉，除非首次出场必要描写
- 声音陈词（脆响、风铃）→ 删掉，或用更具体的声音替代
- 动作陈词（踉跄）→ 用更具体的动作描写替代

### 明喻使用限制

一段话中"好像""似的""如同""仿佛"不超过 2 个。优先用直述或换比喻载体。

```
🚩 超标：
   他的速度快得好像闪电，力量如同山岳，仿佛一头远古巨兽。

✅ 合规：
   他快。快到风声还没追上来，人已经到了。
   一拳。地面裂了。
```

### 重复率控制

同一段内，同一个名词/动词不重复超过 2 次（必要的修辞排比除外）。

### 声音描写限制

除非声音是推动情节的关键信息，否则应尽量减少对声音的直接描写。

```
🚩 过度声音描写：
   骨骼断裂的声音清脆刺耳，在空旷的大殿中回荡。
   （如果不推动情节，读者不需要知道声音的质感）

✅ 保留声音描写：
   "咔。"
   那是骨头断的声音。
   苏寒染听到了。
   （声音是关键信息——确认了伤势的严重程度）
```

### 无意义细节禁止

禁止对非关键身体部位或物品进行与剧情无关的下意识动作描写。每个写进去的细节都必须有叙事功能。

```
🚩 无意义细节：
   他的手指无意识地摩挲着茶杯的边缘。
   （如果这个动作不传递任何情感信息、不暗示关系变化，删掉）

✅ 有功能的细节：
   凌绝尘的手指收紧了一下。
   （暗示他在意的、紧张的事情——读者感受到他不是无动于衷）
```

### 剧情反思 / 水字数检查

写完后，必须追加一轮统一反思：

> 如果删掉一句话、一段文字或一个剧情点，不会影响任何主线情节的因果链、信息链、角色关系变化、冲突升级、伏笔回收或章末钩子，那么它就是冗余，应该删掉。

这不是“文风偏好”，而是结构判断。优先检查以下高发项：

- 过度环境描写：删掉后，角色判断、行动选择、危险感知都不变
- 过度外貌描写：删掉后，人物辨识、关系推进、后续互动都不变
- 重复解释：同一信息前文已经通过动作、对白或事件交代过
- 无功能对白：删掉后，冲突、关系、信息没有净推进
- 松散桥段：整段只是填字，不承担推进责任

执行规则：

1. 先完成当前段落或当前场景
2. 再逐句、逐段做删除模拟
3. 能直接删就删，能并入前后动作就不要单独留段
4. 如果某个剧情点整体可删，就不要补文采，直接删掉或回到大纲重排

硬门槛：

- 只要发现明确的“可删不伤主线”内容，这一版就不能算完成
- 如果只是句子或段落冗余，立即删减后再继续
- 如果某个剧情点整体可删，或连续多个剧情点只承担填充功能，必须回到大纲层处理

按使用场景区分：

- 在 `novel-plan` 或 Opus 试写里：可以做到句子 / 段落 / 剧情点级判断
- 在 `novel-write` 里：默认只处理句子、段落、场景内部铺陈级冗余，不在正文阶段擅自删除整个剧情点或重排结构
- 如果 `novel-write` 过程中发现“某个剧情点整体可删”或“需要重排转折才能成立”，不要在正文里硬修，直接标记并回到 `novel-plan --revise`

## 二、网文爽感检查

### 检查项 1：爽感循环是否完整

```
装逼打脸四步走：
   ① 压制（让读者不爽） → ② 升级（继续加压） → ③ 爆发（一口气释放） → ④ 余韵（爽感持续）

🚩 缺失检查：
   - 有没有"压制"环节？（如果主角从头到尾都很顺利，读者没有"憋气"感，爽感会打折）
   - 有没有"围观者反应"？（配角震惊是爽感放大器）
   - 爆发是否够快？（拖泥带水会泄气）
```

### 检查项 2：章末是否有钩子

```
🚩 每章结尾必须检查：
   - 最后 100 字是否制造了悬念/转折/危机/信息差？
   - 读者看到最后一行，会不会忍不住点"下一章"？

✅ 合格的章末：
   老李说了一句话。
   小明愣在了原地。

❌ 不合格的章末：
   小明回到房间，今天发生了太多事，他需要好好休息。
```

### 检查项 3：信息释放是否有节奏

```
🚩 常见问题：
   - 一口气交代完所有设定（设定集式叙事）
   - 所有伏笔在同一章揭示
   - 没有悬念，读者没有"想知道"的欲望

✅ 正确做法：
   - 每章只释放 1-2 个关键信息
   - 重要信息拆成碎片，穿插在动作和对话中
   - 一个答案引出两个新问题
```

## 三、反常规检查维度

### 维度 1：情节可预测性

**具体检查**：读者看到当前段落，能猜到下一段吗？

- 好人遇到坏人 → 打一架 → 好人赢 → **打破**：坏人其实是卧底 → 两人联手
- 主角获得宝物 → 被人觊觎 → 打脸反杀 → **打破**：发现是陷阱 → 将计就计
- 师父布置任务 → 主角完成 → 获得奖励 → **打破**：任务本身就是考验 → 考验的是"敢不敢拒绝"
- 主角受伤 → 休息 → 恢复 → 更强 → **打破**：受伤反而激活了隐藏血脉

### 维度 2：角色反应戏剧性

**具体检查**：角色的反应是"正常人会做的"还是"让人印象深刻的"？

- 被羞辱 → 愤怒 → 憋着 → **戏剧性**：笑了 → "有意思，继续"
- 获得机缘 → 高兴 → 感谢 → **戏剧性**：眉头微皱 → "这东西...不对劲"
- 面对强敌 → 紧张 → 拼命 → **戏剧性**：打哈欠 → "你讲完了吗？"
- 被质疑 → 解释 → 证明 → **戏剧性**：什么都没说 → 用行动回答

### 维度 3：冲突激烈程度

**具体检查**：这个冲突能不能再升级一档？

- 口角争执 → **翻倍**：当众打脸 → 生死立约
- 切磋比试 → **翻倍**：废掉修为 → 结下血仇
- 利益冲突 → **翻倍**：牵连家族 → 灭门危机
- 任务竞争 → **翻倍**：暗杀 → 背后牵扯宗门高层博弈

### 维度 4：转折意外性

**具体检查**：这个转折在同类小说中常见吗？

- 敌人变朋友 → **意外**：朋友其实一直在利用你
- 弱者突然变强 → **意外**：因为献祭了自己的亲人
- 死前说出秘密 → **意外**：说的秘密是假的，真的秘密藏在别处
- 师父出现救场 → **意外**：但师父是来杀主角的
- 反派内讧 → **意外**：两边都在演戏，真正的主谋在旁观

## 三·五、质量评估维度

除了 AI 味和反常规检查外，从以下维度评估文本质量。**与追求"爽感"冲突时，爽感优先。**

### 1. 感官细节丰富度

检查是否有视觉之外的感官描写（听觉、触觉、嗅觉、味觉）。纯视觉描写 = 平面，多感官 = 立体。

```
🚩 纯视觉：
   凌绝尘站在月光下，白灰色的头发在风中飘动。

✅ 多感官：
   风里有血腥味。凌绝尘站在月光下，白灰色的头发贴着脸颊，
   手上还攥着断了一截的剑鞘。指节发白。
```

### 2. 角色对话独特性

隐去名字后，能否分辨谁在说话？如果两个角色的台词互换后没有违和感 → 不够有个性。

### 3. 角色塑造一致性

角色的言行是否与其底层逻辑一致？反派的立场是否合理？主角的弱点是否在关键时刻自然暴露？

### 4. 意境与主题匹配度

场景描写是否服务于故事当前的氛围？战斗前写鸟语花林 vs 战斗前写鸦雀无声——后者匹配紧张主题。

### 5. 语境适配度

修真世界的词汇、度量、器物是否自洽？是否混入了不协调的现代/系统/编程词汇？

### 6. 场景衔接连贯性

场景转换是否流畅？是否有突兀的跳切让读者"出戏"？

```
🚩 生硬跳切：
   凌绝尘收了剑，转身离开。
   第二天早上，苏寒染在食堂吃早饭。

✅ 流畅衔接：
   凌绝尘收了剑，转身离开。
   剑鞘磕在地上的声音，很轻。
   苏寒染站在原地，直到脚步声彻底消失。
   食堂的粥已经凉了。
```

## 四、正面模板库

### 模板 1：角色出场

```
✅ 有排场：
   门被推开。进来的人没有说话，只是扫了全场一眼。
   原本嘈杂的大殿瞬间安静。
   张师傅来了。

✅ 有反差：
   进来的是个小姑娘。
   十二三岁的样子，扎着两个丸子头，手里拿着一根棒棒糖。
   陈师傅看到她的瞬间，后退了一步。
   "您怎么来了？"

✅ 有悬念：
   小明注意到角落里多了一个人。
   之前没有。
   什么时候来的？没有人知道。
```

### 模板 2：实力展示

```
✅ 对比法：
   小华修炼了三十年才达到的境界。
   小明用了三年。

✅ 画面法：
   他一拳轰出。
   空气被撕裂。
   五丈外的石壁上，多了一个拳头大小的凹坑。

✅ 专家鉴定法：
   "凝气境三重……不，四重！"
   一位长老的声音在颤抖。
   "这不可能！他才修炼了多久？！"
```

### 模板 3：反派压迫

```
✅ 无视型：
   小华没有怒吼。他只是看着小明，眼神像在看一只蚂蚁。
   然后他笑了，笑得很温柔。
   "继续努力。"他说，"我等着你长大。"

✅ 暴力型：
   "你要不要脸？"
   "脸？"小华笑了。
   他抬手，一掌拍向小明的面门。
   没有任何预兆。

✅ 城府型：
   "苏师弟天赋不错。"
   他笑了笑，笑容很温和。
   但小明注意到，他说"天赋"的时候，停顿了一下。
```

### 模板 4：情绪转折

```
✅ 压抑到爆发：
   小明一直在笑。
   从进门开始就在笑。
   笑得很温和，很客气，像是在跟老朋友打招呼。
   直到小华说出了那句话。
   他不笑了。
   "你说什么？"他的声音很轻，"再说一遍。"

✅ 绝望到希望：
   他跪在父亲墓前，已经跪了三天。
   第四天早上，他的手心开始发烫。
   那块玉佩，亮了。
```

### 模板 5：信息揭示

```
✅ 对话揭示：
   "你知道你爹是什么人吗？"
   小明愣了一下。"工厂工人，怎么了？"
   老李笑了。那个笑容里没有温度。
   "工厂工人。"
   他重复了一遍，像是在品味这个词。

✅ 行动揭示：
   小明打开盒子。
   盒子里是空的。
   但他笑了。
   "果然。"

✅ 反转揭示：
   "你父亲，没有死。"
   小明觉得自己的耳朵出了问题。
   "你说什么？"
   "他没有死。"老李看着他的眼睛，一字一顿，
   "但他想让你以为他死了。"
```

## 五、风格示例库（扩充版）

### 示例 1：打脸要狠

- 平淡：小明一掌将小华击飞，冷冷道："不自量力。"
- 大胆：小明没有动。小华的拳头停在他鼻尖三寸处。不是打不到，是被两根手指夹住了。"就这？"小明问。小华的脸涨成了猪肝色。

### 示例 2：悬念要够

- 平淡：小明打开盒子，里面是一颗丹药。
- 大胆：小明打开盒子。盒子里是空的。但他笑了。"果然。"他说，"师父在考验我。"考验什么？他没有说。

### 示例 3：反转要快

- 平淡：小明以为小华是敌人，但小华其实一直在暗中帮他。
- 大胆：小明以为小华是敌人。小华确实在帮他。但帮他的原因，是想借他的手，杀掉自己的竞争对手。等小明反应过来时，刀已经握在他手里了。

### 示例 4：角色出场要亮眼

- 平淡：一个老者走了进来，他是宗门长老。
- 大胆：门被推开。进来的人没有说话，只是扫了全场一眼。原本嘈杂的大殿瞬间安静。张师傅来了。

### 示例 5：冲突升级要快

- 平淡：两人言语不合，决定比试。
- 大胆："你要不要脸？" "脸？"小华笑了，"在这个世上，脸是最不值钱的东西。" 他抬手，一掌拍向小明的面门。没有任何预兆。

### 示例 6：反派要有格调

- 平淡："你死定了！"小华怒吼。
- 大胆：小华没有怒吼。他只是看着小明，眼神像在看一只蚂蚁。然后他笑了，笑得很温柔。"继续努力。"他说，"我等着你长大。"

### 示例 7：心理活动要短

- 平淡：小明心中思绪万千，他想起了师父的话，想起了自己的目标，想起了这一路走来的艰辛。他知道，现在不是退缩的时候。
- 大胆：小明没有多想。他只有一句话在脑海里：不能退。

### 示例 8：结局要留白

- 平淡：小明看着小华离去，心中暗暗发誓，一定要变得更强，超越他。
- 大胆：小华走了。小明没有发誓，也没有咬牙。他只是站在那里，很久。然后转身，继续练剑。

### 示例 9：战斗要有节奏

- 平淡：两人激烈交手，你来我往打了数十招，最终小明找到破绽，一剑刺出。
- 大胆：小华出手了。快。比小明预想的还要快。他侧身堪堪避开，衣角被撕裂。来不及喘息，第二招已到。小明咬牙硬接。虎口震裂。好强。但还不够。

### 示例 10：伏笔要自然

- 平淡：小明注意到老李一直盯着他的胸口看，那里藏着父亲留给他的玉佩。
- 大胆：老李的目光在小明胸口停留了一瞬。很短，短到小明几乎以为是错觉。但他注意到了。

### 示例 11：环境要暗示

- 平淡：天空阴沉，乌云密布，似乎预示着即将到来的风暴。
- 大胆：没有风。一丝都没有。灵药园里的灵草纹丝不动。小明的汗毛竖了起来。

### 示例 12：对话要有信息差

- 平淡："我师父已经死了。"小明说。"我知道。"老李回答。
- 大胆："我师父已经死了。"小明说。老李沉默了一会儿。"是吗。"他端起茶杯，"那可真遗憾。"语气平淡得像在说别人的事。

## 六、集成方式

### 在 novel-plan 中的集成

Event 设计时，对每个关键剧情点进行 AI 味预检 + 反常规检查：

```
📝 Event 设计 - novel-master 检查

当前剧情点：小华抢夺灵草，主角反击

**第一步：AI 味预检**
   ⚪ 情节是否太"AI 式均衡"？无
   ⚪ 角色动机是否太理性？无
   ⚪ 冲突是否不够极端？需注意

**第二步：反常规检查**

❌ 当前设计：
   小华想抢灵草 → 主角提前发现 → 反杀打脸

⚠️ 问题：太可预测，读者能猜到

💡 更大胆的选择：
   方案 A：小华根本没想抢，他是来帮主角的——但帮的方式是"杀了主角"
          （借刀杀人：让主角"死"一次，躲过更大危机）
   方案 B：主角知道小华要来，故意设局——但小华将计就计
   方案 C：两人争夺时，第三方出现，两人被迫联手——但第三方其实是主角安排的

**第三步：剧情反思 / 水字数检查**
   ⚠️ 删除模拟：如果拿掉这个剧情点，主线会不会照样成立？

   - 如果会 → 这个剧情点是填充，删掉或并入前后剧情点
   - 如果不会 → 保留，并明确它承担的推进责任

   例：
   ❌ 当前桥段只是在矿洞外看风景、感叹气氛，删掉后追踪和发现封印阵法照样成立
   ✅ 处理：删掉风景段，把“异常灵气刺痛主角手背”并入进入矿洞前的动作

请选择或提供你的想法：
```

### 在 novel-write 中的集成

`novel-write` 默认只使用 `novel-master` 的**轻量文本质检子集**，不在正文阶段重做大纲级决策。

正文阶段可直接处理的只有：

- AI 味检测
- 总结式结尾
- 解释性对白
- 不必要的过渡段
- 修辞堆砌
- 过度环境 / 外貌描写
- 句子、段落、场景内部铺陈级的水字数

正文阶段**不直接处理**：

- 情节可预测性的大改
- 冲突升级一档
- 转折重排
- 角色核心行为逻辑重写
- 删除整个剧情点

如果检查发现的问题已经超出文本层，应停止在正文阶段硬改，标记并回到 `novel-plan --revise`。

写作时，对关键段落进行实时检查：

```
✍️ 章节写作 - novel-master 检查

当前段落：
   小明心中愤怒，但知道现在不是发作的时候。
   他深吸一口气，转身离去。

**AI 味检测**
   🚨 检测项 6：情感标签替代行为描写
      "心中愤怒" → 用行为展示

**文本质检子集**
   ⚠️ 这段里“他心里很乱”“晚霞像血一样铺满天边”都不承担净推进
   ⚠️ “他深吸一口气，转身离去”是常见过渡句，可以更直接

**剧情反思 / 水字数检查**
   ⚠️ 删除模拟：删掉这两句“心里很乱”和回身看晚霞，主角的受辱、克制和后续反击计划都不受影响
   ✅ 处理：删掉，直接切到他开口或行动

   ⚠️ 删除模拟：删掉这段外貌描写后，人物仍然能通过动作和台词被辨认
   ✅ 处理：压缩成一个功能性记忆点，不保留整段铺陈

**如超出文本层**
   ⚠️ 如果你发现“这场冲突本身不够狠”或“这个转折应该整个重排”，不要在正文阶段硬改
   ✅ 处理：记录问题，回到 novel-plan --revise

请选择：
```

## 检查时机

### novel-plan 检查时机

1. **爽点类型选择后** - 检查是否选择了最常规的类型
2. **起承转合设计时** - 每个阶段检查是否有更大胆的设计
3. **情绪曲线设计时** - 检查情绪起伏是否够大
4. **角色行为设定时** - 检查角色动机是否太"理性均衡"
5. **剧情点切分后** - 逐个问“删掉这个剧情点，主线还成不成立？”

### novel-write 检查时机

1. **心理描写段落** - 检查是否过度 + 是否用了情感标签
2. **对话段落** - 检查是否过度客气 / 解释性过强
3. **过渡段** - 检查是否可以直接切入下一场景
4. **环境 / 外貌描写段** - 检查是否真的承担叙事功能
5. **章节结尾** - 检查是否有钩子 + 是否是总结式结尾
6. **每个场景写完后** - 做一轮句子 / 段落级删除模拟，删掉不影响主线的铺陈
7. **发现结构级问题时** - 不在正文阶段硬改，回到 `novel-plan --revise`

---

## 检查原则

1. **AI 味优先** - 先消除 AI 痕迹，再追求戏剧性
2. **提供选择，而非强制** - 给 2-3 个替代方案，用户可以选择接受或跳过
3. **针对性强** - 检查要具体到当前段落，引用原文
4. **有示例** - 每个检查都要有"平淡 vs 大胆"的对比
5. **尊重创作意图** - 如果用户有意选择"平淡"，不强制要求"大胆"
6. **正面引导** - 不仅指出问题，更提供具体的正面模板
7. **正文不重做大纲** - `novel-write` 只处理文本层问题；结构层问题回到 `novel-plan`
