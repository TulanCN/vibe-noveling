# fuck-it：单章加戏器

## 何时使用

- 【必须触发】用户说：`fuck it`、`fuck-it`
- 用户明确要求把这一章写得更戏剧、更夸张、更有表演力、更有漫画感
- 用户嫌这一章过程太平、太顺、太像普通过桥，但不想改章末结果
- 当前可能处在 `novel-discuss` 的剧情讨论阶段，也可能已经在 `novel-plan` 里落单章大纲

## 调用边界

- 主要给 `novel-discuss` 和 `novel-plan` 用
- 这个 skill 只处理**单章节内部**的冲突升级、表演强化、场面放大和节奏加压
- 这个 skill **不改变单章节的结束目标**
- 如果用户想把后续剧情整体炸开，改用 `booming`
- 如果用户想改的是全书 / 分卷 / arc 的未来走向，改用 `novel-discuss` 或 `novel-bookplan`
- 如果目标是正常落本章大纲，仍由 `novel-plan` 负责
- 如果目标是直接写正文，改用 `novel-write`

## 目标

这不是改章末目标的重规划器，而是同终点前提下的强过程放大器。

它的任务是：

1. 锁死当前章节原本要抵达的结束目标
2. 只放大达到这个目标之前的阻力、烈度、压迫、表演和场面感
3. 默认给出 3 套方案，而且 3 套都必须抵达同一个章末结果

每次触发，默认给出 3 套方案。

## 硬约束

### 1. 先锁死章末目标，再加戏

开始输出前，先用一句话复述当前章的既定收尾。

- 这句话必须写成：`本章结束目标（固定）`
- 后面所有方案都必须共享这个目标
- 不能偷偷把章末结果换成另一个状态
- 不能把本章该完成的事拖去下一章，假装“更有张力”
- 也不能把原本下一章的兑现提前挤进来，导致本章终点被改写

### 2. 只炸过程，不炸终点

允许升级的东西：

- 外部压迫
- 对抗烈度
- 公开场面
- 群体围观
- 误会与试探
- 身份暴露风险
- 时间压力
- 身体代价
- 情绪爆发
- 角色的气场、反应、姿态、语言锋芒

不允许改掉的东西：

- 本章结束时谁赢谁输的既定结论
- 本章结束时主角拿没拿到本章该拿到的结果
- 本章收尾要落在哪个目标节点

### 3. 先过一遍内置的 15 种单章加戏方向，再从中选择 3 种

默认仍然只输出 3 套方案，但这 3 套不能固定写成同一组三板斧。

- 每次都先过一遍内置的 15 种单章加戏方向
- 先从 15 种方向里筛选最适合当前章节的候选
- 再从中选择 3 种最适合当前章的方向输出
- 不要默认固定只用前三种
- 选出来的 3 种方向必须明显不同，不能只是换标题

内置 15 种单章加戏方向：

1. **压迫升级型**：把威压、阻力、代价和不适感往上抬
2. **公开处刑型**：把原本私下的对抗强行搬到众目睽睽之下
3. **表演反杀型**：让主角靠姿态、台词、动作和气场完成反压
4. **身份掀牌型**：把隐藏身份、底牌或真实压差逼到台前
5. **误会爆燃型**：用错误判断和错位理解把场面迅速点燃
6. **失控连锁型**：让一个小动作触发整段局势的连续失控
7. **倒计时逼杀型**：用时限、催命、追兵或硬期限逼场面提速
8. **围观哗然型**：把群众反应、旁观目光和场外评价变成压力器
9. **赌注加码型**：在冲突中途突然把输赢代价抬到更高档位
10. **资源断供型**：让主角仰赖的工具、盟友、退路或体面突然失灵
11. **旧账追魂型**：把旧败、旧债、旧仇或旧承诺拖回当下场面
12. **错位登场型**：让不该此刻出现的人或事强行撞进现场
13. **规则翻脸型**：把原本中立的规则、秩序或程序突然变成武器
14. **诱饵钓杀型**：用引诱、试探和故意放口子逼对手先露破绽
15. **带伤硬撑型**：让主角在挂彩、失态边缘或透支状态下硬顶过去

### 4. 每套方案都必须有明确漫画感

三套方案可以在气质上不同，但不能出现“只有一套有镜头感，另外两套只是普通强化”的情况。

- 每套方案都必须给出能落到场面的漫画感装置
- 每套方案都必须有可视化、可表演、可放大的瞬间
- 每套方案都必须让读者感到“这一幕被故意抬高了”
- 差别只在于漫画感的着力点不同，可能偏压迫、偏掀牌、偏失控、偏规则翻脸或偏带伤硬撑
- 不允许把其中一套写成只有结构加强、没有场面感的平推版

允许你使用：

- 夸张的公开场合
- 强烈的视觉反差
- 突然掀开的身份压差
- 群体围观、窒息式沉默、哗然、失语
- 连锁误判、强行对峙、当场翻盘
- 带有“镜头感”的动作、台词、停顿和表情

但这些都必须服务同一个章末目标，而不是为了酷而改写终点。

## 上下文读取

### 1. 先读最小锚点

优先读取：

1. 最近对话里用户刚给出的本章困局和希望放大的点
2. 当前章已经确定的任务卡、收尾点或章节目标
3. 如果当前在 `novel-plan`，优先读取当前章的 `大纲.md` 或当前对话里已经确认的章节任务
4. 如果当前在 `novel-discuss`，优先读取最近讨论中已经形成的本章目标
5. 上一章真正落地的收尾动作，只读与本章起手直接相关的部分

默认不要全量扫项目；你需要的只是：

- 这一章原本要落到哪里
- 谁在阻拦主角
- 哪个关系最适合加压
- 哪个公开场面最适合放大

### 2. 如果缺的是“章末目标”，先补 1 个问题

这个 skill 的硬边界就是“终点不变”。

所以如果上下文里连本章既定结束目标都没有，允许你先追问 1 个最关键的问题：

- “这章原本必须落到哪个结果？”

除了这个锚点，默认不要先追一长串问题。

## 加戏方法

### 1. 优先放大阻力，不优先改目标

先找最值得加压的入口：

- 主角马上要拿到的成果
- 主角最不能在这章失态的人面前
- 反派最适合公开发难的时机
- 当前关系里最容易爆火星的对手戏
- 刚建立的安全感或体面

### 2. 让“表现力”变成剧情推进器

不要只写“冲突更大了”。

要明确：

- 谁先失态
- 谁先逼近
- 谁在场面上压住了谁
- 谁的语言、动作、停顿、眼神让场面突然抬级
- 哪个反应能让主角的锋芒、狼狈、硬撑或反杀更有戏

### 3. 漫画感要服务推进链

好的漫画感不是单纯夸张，而是让一连串动作更有“啪地一下抬起来”的感觉。

可以考虑：

- 公开处刑式场合
- 本来想低调，结果被强行拽到聚光灯下
- 群体视线突然集中
- 原本藏着的底牌被逼得提前亮相
- 一句台词把整个场子压住
- 一次误判把对抗直接推到更高台阶

### 4. 代价可以重，但范围留在本章内

允许出现：

- 挂彩
- 丢脸
- 误伤
- 信任裂痕
- 资源损耗
- 局部关系恶化

但这些代价不能把本章既定终点改掉，也不要顺手把长期主线重写成另一条线。

## 输出格式

默认按下面的结构输出 3 套方案。

```text
🎭 方案 A｜标题
- 所选方向：
- 本章结束目标（固定）：
- 加戏核心：
- 冲突升级链：
- 角色表现力放大点：
- 漫画感 / 场面感装置：
- 局部代价或挂彩：
- 为什么终点没变但过程更炸：
- 最适合落成的场景节拍：

🎭 方案 B｜标题
...

🎭 方案 C｜标题
...
```

其中必须明确给出：

- `所选方向`
- `本章结束目标（固定）`
- `冲突升级链`
- `角色表现力放大点`
- `漫画感 / 场面感装置`
- `每套方案都必须有明确漫画感`
- `为什么终点没变但过程更炸`

## 落地方式

用户确认其中一条后：

- 如果当前在 `novel-discuss`，再交给 `novel-plan` 落正式大纲
- 如果当前就在 `novel-plan`，直接把选中的方案收束进当前大纲

它不是替代 `novel-plan` 的结构器，而是给单章结构加戏、提气、上强度的分支工具。
