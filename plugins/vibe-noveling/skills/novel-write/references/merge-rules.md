# 合并默认规则

本文件是 `novel-write` 第六步"按剧情点逐点择优合并最终稿"的默认规则来源。

## 风格优先规则

1. **线索推进、群像配合、章末钩子优先卖报小郎君风**：当前剧情点需要线索组织和阅读推进力时，优先选择卖报小郎君版本
2. **环境克制、动作简洁、少修辞优先海明威风**：当前剧情点需要冷硬质感和直写效果时，优先选择海明威版本
3. **正面冲击、速度感、临场决断优先会说话的肘子风**：当前剧情点需要现代快节奏战斗、危险逼近和干净利落的判断反馈时，优先选择会说话的肘子版本
4. **布局交锋、信息差、战斗推演优先大仲马风**：当前剧情点需要戏剧交锋和攻防层次时，优先选择大仲马版本
5. **即时反应、短吐槽、口语提气优先吐槽口语风**：当前剧情点需要人物活气、主角反应和短促接话时，优先选择吐槽口语风版本

## 合并约束

1. 如果角色活气不够、信息已经对了但读起来发平、缺少即时反应，就自动从吐槽口语风中抽取当前剧情点作为优先候选来源
2. 不能把紧张场面卸力成段子腔：吐槽口语风只能提气，不能为了热闹把危险、悬念和冲突写轻
3. 每个剧情点只选 1 个主来源版本：不要把多个版本句群拼接成一个平均化段落
4. 若 5 个版本都不理想，只选最接近目标的一个做局部修字：允许定点改句，但不允许重新变回跨版本拼贴
5. 择优时优先选择更接近当前项目已经人工调整过的正文气质的版本：不要为了整齐把人物活气和网文推进力洗掉
6. 优先保留主角锋芒、主角被区别对待和打脸效果：如果几个版本信息量接近，优先选择更能放大主角特殊性和压制反转的版本
7. 默认压缩与主角装逼链条无关的安全句：不为了平稳而保留那些既不增强主角锋芒、也不推进兑现的概述句
8. 必须在最终稿剧情点标题中标记来源风格：例如 `【剧情点07：雨巷试探｜来源：大仲马】`
9. 允许保留承担推进功能的直给解释：若解释承担剧情信息、行动决策、战术理解或关系变化，不要机械删掉
10. 所有剧情点合并后仍要保持 20 个标题完整可核对：不能在最终稿里再把它们抹平成无标题正文

## 最终稿偏好

最终稿不是五种风格的平均值。

最终稿应更接近当前项目已经人工调整过的正文气质，而不是追求五稿平均、四平八稳的合并腔。

优先保留主线压强、人物活气和网文阅读推进力：

- 优先保留主角锋芒
- 优先保留主角被区别对待
- 倒计时、任务压力、危险逼近、材料线索和主动决策优先保留
- 能让人物更鲜活的口语句、短反应、短态度，优先于平整但发干的概述句
- 能形成打脸效果、身份反差和态度翻转的段落优先保留
- 解释可以更直给一点，但只在它承担剧情信息、行动判断、战术理解或关系推进时保留

允许的提气方式：

- 允许角色即时反应、短促吐槽和带火气的接话
- 允许少量俏皮叙述和轻度作者式调侃
- 允许保留承担推进功能的直给解释

但必须守住边界：

- 吐槽要贴着人物、情境和压力来，不能像段子单独悬空
- 调侃要服务可读性和情绪温度，不能跳出当前 POV 炫机灵
- 默认压缩与主角装逼链条无关的安全句
- 不要把最终稿抹平成一个过分安全、过分中性的合并腔
