# 合并默认规则

本文件是 `novel-write` 第六步"按剧情点逐点择优合并最终稿"的默认规则来源。

## 宪法对齐

合并时必须读取 `CLAUDE.md` 中的"小说宪法"。宪法是风格择优的首要依据，取代任何固定的风格映射规则：

- 根据"叙事纹理"、"作者人格"和"风格锚点"判断当前剧情点哪个版本最接近宪法气质
- "灵魂"和"爽感公式"决定哪些内容优先保留、哪些可以牺牲
- "绝对红线"是不可逾越的底线——任何被选中的版本如果触犯红线，必须做局部修字
- "默认规则"和"默认爽点循环"在宪法未覆盖时作为退回层

## 密度内容约束

密度标记从 `写作约束.md` 的详略约束中读取。合并时，密度影响内容取舍：

1. **低密度剧情点合并时主动砍**：环境描写、配角心理、微动作、过场废话，即使被选中的版本里有也删掉
2. **高密度剧情点不压缩**：如果某个版本在这个剧情点上展开了感官、反应和冲击，不要为了字数均衡而砍掉

## 合并约束

1. 默认优先选择会说话的肘子版本作为主来源：肘子是底稿风格，负责节奏推进、临场判断和动作链
2. 高光剧情点（战斗、对峙、布局交锋、戏剧反转）优先选择大仲马版本
3. 每个剧情点只选 1 个主来源版本：不要把多个版本句群拼接成一个平均化段落
4. 若 2 个版本都不理想，只选最接近目标的一个做局部修字：允许定点改句，但不允许重新变回跨版本拼贴
5. 择优时优先选择更接近小说宪法气质的版本：不要为了整齐把人物活气和网文推进力洗掉
6. 优先保留主角锋芒、主角被区别对待和打脸效果：如果几个版本信息量接近，优先选择更能放大主角特殊性和压制反转的版本
7. 默认压缩与主角装逼链条无关的安全句：不为了平稳而保留那些既不增强主角锋芒、也不推进兑现的概述句
8. 必须在最终稿剧情点标题中标记来源风格：例如 `【剧情点07：雨巷试探｜来源：大仲马】`
9. 允许保留承担推进功能的直给解释：若解释承担剧情信息、行动决策、战术理解或关系变化，不要机械删掉
10. 所有剧情点合并后仍要保持 20 个标题完整可核对：不能在最终稿里再把它们抹平成无标题正文

## 最终稿偏好

最终稿不是两种风格的平均值。

最终稿应以会说话的肘子为底，在战斗、对峙、布局交锋等高光剧情点引入大仲马的戏剧张力和信息差。最终稿应更接近小说宪法定义的作者人格和叙事纹理，而不是追求两稿折中的合并腔。

优先保留主线压强、人物活气和网文阅读推进力：

- 优先保留主角锋芒
- 优先保留主角被区别对待
- 倒计时、任务压力、危险逼近、材料线索和主动决策优先保留
- 能让人物更鲜活的口语句、短反应、短态度，优先于平整但发干的概述句
- 能形成打脸效果、身份反差和态度翻转的段落优先保留
- 解释可以更直给一点，但只在它承担剧情信息、行动判断、战术理解或关系推进时保留

允许的提气方式：

- 允许角色犀利反应、冷面反讽和精准吐槽
- 允许少量俏皮叙述和轻度作者式调侃
- 允许保留承担推进功能的直给解释

但必须守住边界：

- 吐槽要贴着人物、情境和压力来，刀口朝虚伪和荒诞，不能像段子单独悬空
- 调侃要服务可读性和情绪温度，不能跳出当前 POV 炫机灵
- 默认压缩与主角装逼链条无关的安全句
- 不要把最终稿抹平成一个过分安全、过分中性的合并腔
