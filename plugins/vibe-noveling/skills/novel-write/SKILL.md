# 章节创作

本章负责一件事：**把场景链展开为具象的、可阅读的小说正文。** 补环境、感官、动作链、台词；保持角色声线和设定一致；避免 AI 腔。

## 第一步：准备

1. 读当前事件卡 `memory/future/40-events/` 对应文件，提取本章场景链
2. 读上一章 `正文.md` — 确认承接状态和章末钩子
3. `memory/past.md` — 最近摘要和当前状态 → **停，不展开**
4. 读 CLAUDE.md 获取创作风格基线

设定文件缺失就先补全再写作，不自己编造。

## 第二步：写作前强制反思

在动笔前，在 thinking 中完成以下反思。**不输出到对话。**

### 2.1 本章气质（四维定调）

从场景链中感受本章的叙事气质，在四个维度上各选一个极端位——不选中间值：

| 维度 | ← 极端 | 极端 → |
|------|--------|--------|
| **节奏** | 绷紧：句句推进 | 舒展：有留白 |
| **距离** | 冷眼旁观 | 贴着 POV 人物呼吸 |
| **密度** | 锋利：每句推进 | 绵密：感官铺陈 |
| **质感** | 粗粝直接：短句口语 | 精密细腻：长句层次 |

选好后整章写作全程维持。如果不同场景确实需要不同气质，预先声明切换点。

### 2.2 本章最易被 AI 腔污染的位置

基于场景链，预先定位 2-3 个最容易翻车的点，每个给出本章的具体预防措施：
- **情感高危场景** → 预先决定用什么动作或感官细节替代情感标签
- **对话失真高危场景** → 预先决定谁在哪句话撕破脸、打断、反问
- **转场空泛高危位** → 预先确定触发切场的具体动作
- **结尾泄力高危位** → 预先确定留动作还是留意念

### 2.3 本章最该被记住的一个瞬间

选一个具体的、可见的画面、台词或动作。整章写作围绕它组织。
- 必须可见，不是"主角成长了"而是"他做了 X"
- 写这个瞬间时，允许比其他段落多花篇幅

### 2.4 本章禁用句式

根据本章气质和内容，声明 3-5 个具体禁用的表达模式，落到能在写作时直接检查的粒度。

## 第三步：写作

从场景链直接展开为正文。场景链每场是四通道因果语句（`<环境> [动机]（动作）结果`），从中理解推进方向后展开为具象正文。

以第二步的反思结论为内置约束。不插入剧情点编号、字数配额等中间层。风格从 CLAUDE.md 继承。

正文保存到 `chapters/e{event_padded}/ch-{chapter_seq}/正文.md`。

## 第四步：后处理

对正文一次性 review：

1. **一致性校验** — 调用 `consistency-guard`，检查称谓、时间线、承接、设定冲突。局部修补，不重做结构。
2. **AI 味检测** — 读 `references/ai-smell-checklist.md`，10 项逐项检测并修复。
3. **独立润色** — 读 `references/polish-rules.md`，按 5 条原则对整章做文风转换：调整语序、替换同义词、打乱句子结构、弱化精炼度、营造普通作者手感。不大幅改字数，不改变核心表意。

## 输出

```
chapters/e{event_padded}/ch-{chapter_seq}/
└── 正文.md
```

交付后提示用户：标记需修改位置后运行 novel-revise，或确认后运行 novel-sync。

## 禁止

- 擅自改变场景链的本章边界或关键推进链
- 忽略设定缺失自己编造
- 结构问题不在正文阶段硬修——回事件讨论改事件卡
- 借一致性修补名义重做结构
- 在场景链和正文之间插入剧情点列表等中间层
- 将第二步的反思结论输出到对话
