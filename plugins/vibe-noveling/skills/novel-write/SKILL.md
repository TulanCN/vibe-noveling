# 章节创作

## 目标

根据场景化剧情大纲（含上游剧情点结构），并结合上游 Opus 测试报告提炼出的写作约束，按用户选择的 2-3 种写作风格并行创作多个正文版本。每个版本经过校对后交付，由用户手动合并最终版本并通知，随后自动进入同步流程。

**核心特性**：多风格并行输出，不反复讨论，直接给你挑。

## 核心理念

章节写作负责两件事：
1. **把大纲变成血肉** — 在骨架上添加环境描写、心理活动、感官细节，根据角色语言风格（来自角色设定文件）创作台词
2. **提供风格选择** — 同一剧情点用不同笔法写出来，让你挑最好的

- 大纲已经决定了：场景、剧情概要、情绪走向、写作提示
- 正文写作负责：把这些变成"能让人读下去的文字"，包括创作具体台词和场景细节
- 不再反复讨论"怎么写"，而是直接写出来让你选

## 使用方式

```
/novel-write 01    # 创作第 1 章
/novel-write 02    # 创作第 2 章
/novel-write next  # 创作下一章（自动确定编号）
```

## 完整流程

```
  第一步：获取上下文（缓存优先）

       │  ├─ 缓存有效 → 直接复用（跳过收集）
       │  └─ 缓存无效 → 调用 context-collector
       │     收集角色设定、世界观、伏笔状态
       ▼
  第一步 A：设定缺失检测
       │  有缺失 → 补全 → 重新收集
       ▼

       │  从 chapters/ch-xxxx-outline.md 读取剧情点和写作提示
       ▼
  第二步 A：读取 Opus 测试报告（如存在）
       │  检查 ch-xxxx-opus-report.md
       │  ├─ 判定：需先重做大纲 → 停止写作，回到 novel-plan --revise
       │  └─ 其他判定 → 提炼“写作约束摘要”
       ▼

## 执行流程

### 第一步：获取上下文（优先使用缓存）

**在开始任何写作前，必须获取完整上下文。优先使用 plan 阶段生成的缓存，避免重复收集。**

#### 1.1 检查上下文缓存

**首先检查 `chapters/ch-{chapter_padded}-context.md` 是否存在且有效。**

缓存有效性判断：
- 文件存在 → 读取缓存头的 `生成时间`
- 检查相关设定文件的修改时间是否晚于缓存生成时间：
  - 出场角色的设定文件（`memory/entities/characters/`）
  - `memory/worldbuilding.md`
  - `memory/_graph.json`
- 所有文件都未修改 → **缓存有效**，直接使用
- 有文件被修改 → **缓存失效**，重新收集

#### 1.2 根据缓存状态选择路径

**缓存有效**：
```
📖 使用上下文缓存

检测到 chapters/ch-{chapter_padded}-context.md（缓存有效）
生成时间：{timestamp}
设定文件无变更 → 直接复用

跳过上下文收集，节省 token 消耗。
```
直接使用缓存内容作为上下文摘要，跳过 context-collector 调用。

**缓存不存在或已失效**：
```
📖 缓存不可用

{缓存不存在 / 缓存已失效：角色设定文件已变更}

调用 context-collector 重新收集上下文...
```

调用方式：
```
Agent 工具，subagent_type: context-collector
prompt: 为第 X 章收集上下文，大纲文件：chapters/ch-{chapter_padded}-outline.md
```

**context-collector 会自动**：

1. 分析章节大纲，识别需要的设定类型
2. 调用知识图谱搜索相关角色、地点、物品
3. 读取设定文件
4. 组织成结构化的上下文摘要
5. 输出到缓存文件 `chapters/ch-{chapter_padded}-context.md`（供后续复用）

### 第一步 A：设定缺失处理

**如果 context-collector 报告设定缺失，必须先补全设定，再继续写作。**

使用 AskUserQuestion 展示三种处理方案：

```
方案 A：【暂停写作，补全设定】（推荐）
  角色缺失 → /novel-discuss 创建角色
  世界观缺失 → /novel-discuss 构建世界观
  剧情缺失 → /novel-plan

方案 B：【调整正文，绕过缺失】
  修改章节大纲，避免使用未定义的设定

方案 C：【临时标记，后续补充】
  在正文中用 [待设定:XXX] 标记
```

**禁止**：忽略设定缺失、自己编造设定、假设"后续再补"。

### 第二步：读取大纲

从 `chapters/ch-{chapter_padded}-outline.md` 读取本章对应的剧情点、角色语言风格和写作提示。

**剧情点结构来自上游**：novel-plan 的第 7 步设计了剧情点（结构层），并在第 9 步保存为 outline.md。正文写作直接沿用大纲中的剧情点结构。

**大纲关键认知**：
- **剧情点标题** `### 剧情点N：【标题】`：大纲已按剧情点分段，正文按此结构写作
- **场景类型与情绪走向** `场景类型：[类型] | 情绪走向：[走向]`：决定该段的写作策略和节奏
- **角色语言风格**：从角色设定文件（通过 context-collector 获取）获取，创作台词时必须遵循，确保角色声音差异化
- **剧情概要**：2-3句剧情概要给出方向，正文在概要范围内自由发挥台词和场景细节
- **写作提示** `> ✏️`：这是上游给出的落文抓手，不是第二份剧情概要；严格遵循其中的起手动作、焦点细节、对话质地和禁解释项
- **环境描写**：根据地点和氛围信息创作
- **心理活动**：大纲刻意不写心理活动，正文写作的核心任务

**禁止**：大幅改动剧情概要方向、改变剧情点顺序、违背写作提示。

### 第二步 A：读取 Opus 测试报告（如存在）

如果存在 `chapters/ch-{chapter_padded}-opus-report.md`，必须在正式写正文前先读取它。

这份报告的作用不是提供“参考措辞”，而是提供两个东西：

1. **放行判定**：这一章现在能不能直接进入正文创作
2. **写作约束**：正式写作时必须补出、必须避免、必须压缩的点

#### 2A.1 读取规则

优先读取以下部分：

- `## 总体结论`
- `## 写作中暴露的问题`
- `## 被迫脑补的内容`
- `## 回填建议`
- `## 建议修订动作`

#### 2A.2 放行规则

**如果不存在 `chapters/ch-{chapter_padded}-opus-report.md`**：

```text
📎 未找到 Opus 测试报告

本次按 outline + context 直接写作。
风险：不会自动继承上游试写发现的问题点。
```

继续写作，但不享受上游约束闭环。

**如果报告判定为 `需先重做大纲`**：

```text
🚫 暂停正文创作

Opus 测试报告判定：需先重做大纲

原因：
1. 当前结构问题属于大纲层，不适合在正文阶段硬修
2. 如果继续写，会把已知结构问题放大成正文返工

下一步：
   /novel-plan --revise
```

立即停止，不进入风格选择和并行写作。

**如果报告判定为 `需小修后写`**：

先判断报告里的问题是否都能转成**局部写作约束**。

可以继续写的情况：
- 补一个已有剧情点内部的触发动作
- 收紧某段信息投放
- 强化某个角色的声线抓手

必须停止并回到 `novel-plan --revise` 的情况：
- 需要新增剧情点
- 需要删除整个剧情点
- 需要重排剧情点顺序
- 需要改动章节收尾目标
- 需要调整伏笔处理位置

**如果报告判定为 `可直接写`**：

也必须读取报告中的“必须避免重犯的问题”，作为正式写作的结构约束。

#### 2A.3 提炼写作约束摘要

不要把整份 `opus-report.md` 原样塞进写作 agent 的 prompt。必须先压缩成一段短的“写作约束摘要”。

提炼原则：

- 只保留真正影响执行的 3-6 条
- 每条都必须是可操作的动作，不保留抽象诊断语
- 优先提炼两类信息：
  - **必须补出**：桥接动作、触发信息、角色语言抓手
  - **必须避免**：重复解释、松散桥段、无功能对白

输出格式：

```text
🧷 写作约束摘要

来源：chapters/ch-{chapter_padded}-opus-report.md
放行判定：需小修后写

必须补出：
1. 剧情点3结尾补出主角决定继续追查的触发动作

必须避免：
1. 不要把夜市环境铺陈写成独立长段
2. 叶芷恒对白禁止只承担信息搬运功能

若写作过程中仍需要新增剧情点/重排顺序/改收尾目标：
立即停止正文创作，回到 /novel-plan --revise
```

### 第三步：选择写作风格

展示 5 种作家风格选项，用户选择 2-3 种。

```
🎨 选择写作风格

以下 4 种作家风格可供选择，每种风格对应一位代表性作家的语言特色：

1. **海明威风（极简硬派）** - 冰山理论：只写露出来的 1/8，用动作和对话讲故事
   示例：苏寒染推开门。屋里没人。桌上有一封信。他没有立刻拆开。他先倒了杯水，喝了一半，然后才拿起那封信。信封上没有署名。

2. **卖报小郎君风（轻悬疑群像）** - 线索牵引剧情，群像鲜活，轻快里带刀，章末必须留钩子
   示例：门没锁。苏寒染站在门外，看了两息，没进去。"你先。"他说。凌绝尘皱眉。"凭什么？"苏寒染低头，看了一眼门槛边那半枚泥印。"因为里面的人，刚死没多久。"院子里忽然没人说话了。

3. **火星引力风（暴力美学+情感拉满）** - 极致暴力的战斗画面，细腻入微的情感，精准的节奏把控
   示例：凌绝尘的剑落下来的时候，苏寒染的耳朵里什么声音都没有。只有风声。然后他听到了第二声。是剑尖擦着骨头滑过去的声音。凌绝尘退了半步。就半步。苏寒染看着他的眼睛——那双蓝瞳孔里，有什么东西在碎裂。

4. **大仲马风（基督山式精密布局）** - 多线伏笔交织，戏剧性反转，优雅的对话交锋，读者永远比角色少知道一步
   示例：苏寒染在账簿上划掉了第三个名字。他放下笔，端起茶杯。门外传来脚步声——他等了七天的人，终于到了。但他没有起身。他只是微微侧头，对着空无一人的角落说："出来吧，你该看到了。"角落里没有声音。但空气变了。

### 第四步：并行创作 + 校对

为每种选定的风格 dispatch 一个 `general-purpose` Agent，**并行**写作。

**调用方式**：在一条消息中发出多个 Agent 工具调用，实现并行。

**关键：每个 Agent 的 prompt 中必须包含明确的输出文件路径**，格式为 `chapters/ch-{编号}-{风格简称}.md`。
例如第 1 章的大仲马风版本必须写入 `chapters/ch-0001-大仲马.md`，海明威风写入 `chapters/ch-0001-海明威.md`。
**禁止让 Agent 自行选择文件路径或输出到同一文件。**

每个 Agent 的任务：
1. 先读取并遵守上游的“写作约束摘要”
2. 按剧情点顺序写完整章节（2500-4000 字）
3. 使用指定的写作风格
4. 写完后执行 novel-master 轻量文本质检（AI 味 + 结尾/对白/过渡 + 报告约束复检）
5. 不通过则修改后重新自检（最多 3 轮）
6. **将最终版本写入指定文件路径 `chapters/ch-{编号}-{风格简称}.md`**
7. 输出格式：`【剧情点N：名称】\n(正文内容)`

这里默认**不**用 novel-master 重做结构设计：如果发现需要升级冲突、重排转折、删除整个剧情点，或必须违背 `opus-report` 才能写通，停止在正文阶段硬改，改为标记后回到 `novel-plan --revise`。
但“冗余描述 / 水字数”必须在正文阶段自己清掉，不能把这部分责任推回 `novel-plan`。

## 通用写作要求（所有风格必须遵循）

### 节奏控制
- 短句为主，长句点缀，段落不超过 4 行
- 紧张时一句一行，舒缓时允许长句
- 能删的字全删

### 句式规则
- 禁止使用"不是……，而是……"句式，直接说结果即可
- 对话要有"刺"，不要客气对话
- 越重要的事话说得越少
- 经常被打断，反问比陈述多
- 说一半，留一半
- **角色声音差异化**：严格按照角色设定文件中的语言风格创作台词，确保每个角色说话方式不同

### 视角规则
- 紧贴主角有限第三人称
- 主角不知道的，读者不能知道
- 用感官代替心理描写

### 章末钩子
- 最后 100 字必须有悬念/转折/危机
- 禁止总结式、抒情式、平淡式结尾

## novel-master 轻量文本质检清单（写完后逐条检查）

0. **格式检查**（最高优先级）：是否每个剧情点都有 【剧情点N：名称】 标题？标题前后有空行？没有标题 → 立即补上，不要继续后续检查
0.5. **禁用句式检查**：是否使用了"不是……，而是……"句式？发现任何一处 → 立即改掉（这是强制禁止句式）
1. 过度心理描写 → 删掉 2/3，只留最关键一句
2. 所有人都客气 → 给对话加"刺"
3. 万能形容词 → 用具体动作代替
4. 总结式结尾 → 改成悬念式断句
5. 均衡分配 → 放大重点，压缩过渡
6. 情感标签 → 用行为展示
7. 解释性对话 → 用冲突和行动传递
8. 不必要的过渡段 → 直接切入下一场景
9. 修辞堆砌 → 一段最多一个修辞
10. 报告约束复检 → 是否重犯 `opus-report` 提醒的桥接缺失、信息过密、声线发虚等结构问题？若重犯，立即修改；若无法在不改结构的前提下修复，停止并回到 `novel-plan --revise`

## 成稿后冗余描述清理检查（必须单独执行）

这一步发生在**完整正文已经写完，并且通过基础文本质检之后**。它不是上游 `Opus` 试写报告的延续，而是针对你刚刚写出来的正式正文做第二次清理。

核心判定只有一句：

> 如果删掉一句、一段或一组连续描写后，不影响任何主线因果、信息推进、角色关系变化、冲突升级或章末钩子，就删。

执行顺序：

1. 先冻结结构，不新增剧情点，不重排顺序
2. 再逐段检查当前成稿里的环境描写、外貌描写、解释段、对白和过渡段
3. 优先做三种动作：
   - 直接删除
   - 压缩成一句
   - 并入前后已有动作/对白
4. 清理完后，再快速复查一次节奏和衔接

高发项：

- 过度环境描写：删掉后，角色判断和行动没有变化
- 过度外貌描写：删掉后，人物辨识和关系推进没有变化
- 重复解释：同一信息已经在动作或对白里出现过
- 无功能对白：删掉后，关系、冲突、信息没有净变化
- 松散过渡：只是“看起来像在写”，但不承担推进责任

硬规则：

- 能删整段，就不要只改形容词
- 能压成一句，就不要留三句
- 如果清理后发现某个剧情点几乎被掏空，说明它可能本来就结构不足；此时停止正文阶段硬补，并回到 `novel-plan --revise`

## ⚠️ 输出格式（最高优先级）

**每个剧情点必须以 【剧情点N：名称】 作为标题行，标题行前后各空一行。禁止合并标题、禁止省略标题、禁止写成连续散文。**

这是硬性格式要求，自检时第一条就检查这个。

正确示例：
```
【剧情点1：村庄外围-部署通讯阵】

苏寒染蹲在坑边。面前的泥地上刨了一个浅坑……

（正文内容，100-400 字）

【剧情点2：来修真界三年】

他来修真界三年了……

（正文内容，100-400 字）
```

错误示例（绝对禁止）：
```
❌ 直接写正文，没有剧情点标题
❌ 标题和正文之间没有空行
❌ 多个剧情点混在一起，只用一个标题
❌ 用 ## 或 ### 等其他格式代替 【剧情点N】
```

## 执行步骤

1. 通读剧本、上下文和 `Opus 测试约束摘要`，理解哪些地方必须补、必须避
2. 如果你发现某条约束只能通过新增剧情点、重排顺序或改收尾目标来满足：立即停止并回报
3. **按剧情点顺序逐个写作，每个剧情点必须以【剧情点N：名称】开头**
4. 全部写完后，先检查格式：是否每个剧情点都有标题？（如果没有，立即补上）
5. 逐条执行 novel-master 轻量文本质检
6. 再逐条执行“报告约束复检”
7. 然后单独执行“成稿后冗余描述清理检查”
8. 发现问题立即修改，修改时不要删除剧情点标题
9. 修改后重新自检（最多 3 轮）
10. **通过后，将完整正文写入文件 `chapters/ch-{编号}-{风格简称}.md`**
11. 输出完成报告（包含文件路径、字数、以及是否完全遵守 Opus 约束）

#### 风格指令详情

以下指令会根据用户选择的风格注入到 Agent Prompt 中：

**海明威风（极简硬派）**：
```
风格核心：冰山理论。你只写露出来的 1/8，剩下的让读者自己感受。
- 句式：短句为主，一句一行是常态。绝不用超过两行的句子
- 用词：动词 > 形容词，能一个词解决绝不用两个。永远用最简单的词
- 对话：越短越好，"嗯""不""走"就是完整回答。不说废话
- 心理：不写心理描写。用动作和沉默代替一切内心活动
- 描写：只写看得见听得见摸得着的。不写"感觉""似乎""仿佛"
- 战斗：一招一式，干净利落。每个动词都是武器，不堆砌
- 过渡：直接跳切，不要过渡段。时间流逝用动作暗示，不要"三天后"
- 禁区：任何内心独白、任何"他感到XX"、任何修辞（一段最多一个比喻且尽量不用）、任何抒情
- 经典句式参考："他喝了酒。酒很烈。他没有咳嗽。"
```

**卖报小郎君风（轻悬疑群像）**：
```
风格核心：线索牵引剧情 + 群像撑住场面 + 轻快里带刀。读者要一边读得顺，一边忍不住追下去。
- 句式：中短句为主，节奏清爽。紧张时短句连发，插科打诨时允许一两句口语化长句
- 用词：口语感要自然，但不能油滑。优先使用能直接推动判断的动作词、线索词、身份词
- 对话：群像互怼要鲜活，每个人说话都要有自己的脾气。信息尽量藏在插话、反问、岔开话题里
- 线索：每个场景至少有一个可见线索或异常点，先抛现象，再给半个解释，剩下的留到后文
- 节奏：开场尽快抛问题，中段靠人物互动推进，结尾必须留一个钩子或反转，不能平着收
- 群像：配角不是围观器，也不是统一口径的捧哏。每个人都要贡献态度、判断或麻烦
- 情感：底色要有人味。轻松段落里有默契，危险段落里有克制，不直接煽情
- 爽点：爽感来自破局、识破、反将一军，不只靠正面碾压。主角要像在解题，也像在做局
- 禁区：整段说明背景、把线索一次说透、所有配角都只会喊"震惊"、玩梗过多毁掉氛围
- 经典句式参考："门没锁。苏寒染没进去。他看着门槛边的泥印，问：'刚才谁来过？'没人回答。于是他笑了一下。'那就不是来过。是还没走。'"
```

**火星引力风（暴力美学+情感拉满）**：
```
风格核心：极致暴力的战斗画面 + 细腻入微的情感描写 + 精准的节奏把控。披着爽文外衣的精品。
- 句式：中短句为主，战斗时短句密集但有序，不乱。关键动作单句成段制造冲击
- 战斗：暴力美学——写伤不写"疼"，写声音不写"响"。剑擦着骨头、血喷在脸上、
  耳朵里什么都没有然后听到心跳。画面感极强，像慢镜头拆解每个动作
- 情感：细腻到让读者能"看到"角色的微表情变化。用一个眼神、一个小动作传递千言万语。
  不煽情，但让读者自己受不了。"有什么东西在碎裂"比"他哭了"更有效
- 信息差：善于操控读者的信息量。读者知道的比角色多时，焦虑感翻倍。
  揭示真相前先铺垫一个更可怕的可能性
- 节奏：伏笔丰富但不拖沓。每 300-500 字必须有一个小钩子或情绪转折。
  章末钩子精准——在读者最想知道的时候断开
- 人物：反派不脸谱化，有鲜明个性和合理动机。连跑龙套都有辨识度
- 禁区：战斗场面写成流水账、情感直接说破（用细节暗示）、超过500字没有节奏变化
- 经典句式参考："凌绝尘退了半步。就半步。苏寒染看着他的眼睛——那双蓝瞳孔里，
  有什么东西在碎裂。"
```

**大仲马风（基督山式精密布局）**：
```
风格核心：精密布局式叙事 + 戏剧性反转 + 优雅的对话交锋 + 信息差操控。
  像《基督山伯爵》一样，每个细节都是棋子，读者永远觉得自己猜到了——直到被反杀。

- 句式：中短句为主，布局和揭露时节奏刻意变慢——让读者看清每一步棋。
  反转时节奏突然加快，短句密集。日常场景允许优雅的长句和从容的节奏
- 信息差操控：这是本风格的核心武器。
  - 读者知道角色不知道的事 → 焦虑（"别去！""那是陷阱！"）
  - 角色知道读者不知道的事 → 好奇（"他为什么笑？""那张纸条写了什么？"）
  - 所有人都被蒙在鼓里 → 震撼（当真相揭晓的那一刻，前面所有的细节突然全部串联起来）
  每个场景至少埋一层信息差，揭示一层，保留一层
- 对话交锋：优雅但有杀机。像剑术比赛——表面上彬彬有礼，每一句都在试探、暗示、布局。
  双关语、话中有话、正话反说。角色的真实意图藏在礼貌的措辞下面。
  最致命的威胁用最温和的语气说出来
- 布局感：角色不是在反应，而是在执行计划。每个行动都有目的，每句话都在为三步之后的棋局铺垫。
  读者在阅读过程中逐渐拼凑出全貌——像拼图一样，每次获得一块新碎片都让之前的碎片有了新含义
- 戏剧性反转：反转不是突然的，而是回溯性的。反转发生后，读者回头看前面的段落，会发现线索一直都在。
  "原来他当时那么说，是因为……"——这种恍然大悟的感觉是本风格追求的最高效果
- 角色塑造：每个角色都有自己的秘密和目的。没有纯粹的坏人或好人，只有立场不同的人。
  反派的动机要合理到读者会觉得"如果是我，可能也会这么做"
- 氛围：从容、优雅、暗流涌动。表面平静的水面下是致命的暗流。
  关键场景用仪式感——重要的时刻要有仪式感（特定的时间、地点、物品、人物出现的方式）
- 战斗：战斗不是蛮力对轰，而是智力的延伸。先布局再出手，以最小代价获得最大效果。
  最精彩的战斗是一招制敌——因为前面十步的布局已经决定了胜负
- 禁区：反转没有前文铺垫（必须回溯性成立）、角色只有单一目的、对话直来直去没有潜台词、
  信息差一次性全部释放（要分层剥开）、布局没有回报（前面埋的棋子后面必须收）
- 经典句式参考："他等了七天。不是等一个人。是等一个时机。
  现在，时机到了。他合上账簿，微微一笑。'请他进来。'
  他用了'请'这个字。七年前，没有人对他说过这个字。"
```

### 写作流程中的文件操作

- **最终稿由用户手动合并**：保存到 `ch-{编号}.md`
- **用户通知后自动同步**：收到“已合并 ch-{编号}.md”即执行 `/novel-sync chapter {chapter}`
- **禁止行为**：多个 Agent 写入同一文件路径、Agent 不写文件只在对话输出、使用非标准命名格式

## 关键原则

1. **忠于大纲** — 在剧情概要范围内自由发挥台词和场景细节，不擅自改变剧情方向或删减场景
2. **风格分明** — 每种风格有明显差异，不是微调
3. **质量保证** — 每个版本都经过校对和一致性检查
4. **用户做主** — 最终合并由用户手动执行，不替用户选
5. **遵循提示** — 剧本中的 `> ✏️` 写作提示是落文抓手，不是参考评论；必须遵循
6. **先看报告，再写正文** — `opus-report` 是放行条件和约束来源，不是可有可无的附加说明
7. **约束要提炼后传递** — 传给写作 agent 的是短的“写作约束摘要”，不是整份诊断报告
8. **结构问题不上正文阶段硬修** — 需要新增/删除/重排剧情点时，回到 `novel-plan --revise`
9. **冗余描述在正文阶段清理** — 不把“水字数”主要寄希望于上游试写，正式成稿后必须再删一轮

## 写作风格指南（通用参考）

以下指南适用于所有风格，每种风格会在通用基础上做变体调整。

**核心定位：网文爽文风格，不是纯文学**

记住：读者刷手机看，每句话都要让他想点"下一行"。

### 一、节奏控制

**核心原则：短句为主，长句点缀，段落切碎**

#### 1. 句式节奏

```
✅ 紧张时 - 一句一行，制造紧迫感：
   小明没有说话。
   他只是看着小华。
   看了很久。
   然后笑了。

✅ 舒缓时 - 可以用长句，但不要超过两行：
   晨光透过窗棂，在大殿青石地面投下斑驳光影。弟子们陆续到齐，窃窃私语声渐渐低了下去。

❌ 永远不要这样写：
   小明心中思绪万千，他想起了师父的话，想起了自己的目标，想起了这一路走来的艰辛，
   他知道现在不是退缩的时候，所以他深吸一口气，握紧了拳头，决定不再逃避。
```

#### 2. 段落切分

- **一个段落不超过 4 行**（手机阅读友好）
- **重要信息独立成段**
- **对话和动作交替出现**，不要大段纯对话或纯描写

```
✅ 小明推开门。
   大殿内已经站满了人。
   气氛有些不对。
   所有人的目光都汇聚在同一个方向——高台上的张师傅。
   他的脸色，很差。

❌ 小明推开门，大殿内已经站满了人，气氛有些不对，所有人的目光都汇聚在
   高台上的张师傅身上，他的脸色看起来很差。
```

#### 3. 信息释放节奏

- **关键信息拆成碎片**，穿插在动作和对话中
- **不要一口气交代完设定**
- **让读者自己拼凑真相**

```
✅ "你知道你爹是什么人吗？"
   小明愣了一下。"工厂工人，怎么了？"
   老李笑了。那个笑容里没有温度。
   "工厂工人。"
   他重复了一遍，像是在品味这个词。
   "是。工厂工人。"

❌ "你父亲其实不是工厂工人，他是第一批灵界探索者，十五年前在执行任务时被幽冥教杀害了。"
```

---

> **注意**：爽感设计（打脸节奏、围观震惊、实力碾压、连锁震惊）属于剧情设计能力，已在 novel-plan 的「剧情设计工具箱」中详细定义。正文写作时，大纲中的剧情点已包含爽点类型和情绪走向，按写作提示执行即可。

### 二、对话风格

#### 1. 角色对话要有"刺"

每个角色说话都应该有自己的个性和目的，不要写"礼貌对话"：

```
✅ 有攻击性：
   "你配？"
   只有两个字。小华甚至没有回头。

✅ 有城府：
   "苏师弟天赋不错。"
   他笑了笑，笑容很温和。
   但小明注意到，他说"天赋"的时候，停顿了一下。

✅ 有压迫感：
   门被推开。
   来人没有说话，只是扫了全场一眼。
   嘈杂的大殿，瞬间安静。

❌ "苏师弟，你这样做不太好吧？我认为我们应该好好商量一下。"
```

#### 2. 对话要推动剧情

每句对话必须至少完成以下之一：

- **推动情节**
- **揭示角色性格**
- **制造冲突**
- **埋设伏笔**

```
✅ "你师父临走前，给你留了什么东西？"
   小明心中一跳。
   "你怎么知道我师父？"（制造冲突 + 推动情节）

❌ "今天天气不错啊。" "是啊，天气确实不错。"（纯废话）
```

#### 3. 说一半，留一半

最好的对话不把话说完，让读者去猜：

```
✅ "小心张师傅。"
   "为什么？"
   老李没有回答。他端起茶杯，吹了吹。
   "有些事，你还不需要知道。"

✅ "这个任务……"
   小华顿了顿，嘴角微微上扬。
   "有点意思。"
```

### 三、视角与叙事

#### 1. 紧贴主角的有限第三人称

- **永远从主角的视角出发**
- **主角不知道的，读者也不能知道**
- **用主角的感官去描写世界**

```
✅ 主角视角：
   小明闻到了血腥味。
   很浓。
   他下意识握紧了剑柄。

✅ 主角不知道的事，用"感觉"而非"全知"：
   老李的表情变了。具体怎么变的，小明说不上来。
   但他有一种直觉——
   这个老头，在试探他。

❌ 全知视角（跳到其他角色脑子里）：
   小华心中暗想，这个小明不过是个废物，今天一定要让他好看。
   （除非切换视角章节，否则不要这样做）
```

#### 2. 感官描写优先于心理描写

不要大段内心独白，用感官体验代替：

```
✅ 小明的手心全是汗。
   他攥了攥拳头，又松开。
   "没事。"他说。声音比自己预想的要稳。

❌ 小明心中充满了矛盾和纠结，他一方面觉得自己应该勇敢面对，
   另一方面又害怕失败后的嘲笑，这种复杂的情绪让他一时间难以做出决定。
```

---

### 四、章末钩子

**每章结尾必须有一个让读者忍不住点"下一章"的理由。**

#### 钩子类型：

- 悬念型：抛出问题但不给答案（老李说了一句话。小明愣在了原地。）
- 转折型：最后一句推翻前面的认知（盒子里是空的。但小明笑了。"果然。"）
- 危机型：在最紧张处断开（剑光一闪。小明没有躲。）
- 信息型：透露一个关键信息（"你父亲，没有死。"）
- 反转型：预期与结果相反（他以为自己必死无疑。然后他听到了一声轻笑。）

#### 章末禁忌：

```
❌ 总结式结尾：小明望着远方，他知道，这只是开始。
❌ 抒情式结尾：月光洒在少年身上，映照着他坚毅的脸庞。
❌ 平淡式结尾：小明回到房间，今天发生了太多事，他需要好好休息。
```

### 五、男频网文风格要点

以下为通用基线，各种风格在此基础上做变体调整。

**核心信条**：只写发生的事，不解释为什么。冰山水面下的 7/8 就是伏笔和悬念。

**句式**：

- 极短句为主，紧张时一句一行，关键动作独占一段
- 舒缓场景允许长句，但一段不超过 3 句
- 能删的字全删，一个形容词解决的绝不用两个

**用词**：

- 动词 > 形容词（"砸"比"用力地打"好）
- 具体名词 > 抽象概念（"拳头"比"攻击"好）
- 数量词和颜色词是武器（"三寸""两杯""红色"）
- 永远用最简单的词（"说"比"厉声道"好）

**对话**：

- 短，经常被打断，越重要的事话说得越少
- 反问比陈述多，讽刺比客气多
- 信息藏在缝隙里，角色说的和想的不是一回事
- 经常用动作代替回答（沉默、点烟、摇头）

**战斗**：

- 重视速度感和力量感，用对比制造压迫感
- 受伤不喊疼，恐惧不颤抖——用冷静克制制造力量感

**修真体系**：

- 实力等级贯穿叙事，每次突破有具体描写
- 资源争夺用具体数字（灵石、丹药、材料）
- 围观者的震惊反应是爽感放大器

**大场面**：

- 短句堆叠制造史诗感
- 反派出场不说话就有气场

**心理描写**：

- 用身体反应代替（手心出汗、喉咙发紧）
- 偶尔一句极短独白（"不能退。"）
- 从不解释为什么有这个感受

### 六、AI 味消除清单

写完正文后，逐条检查并修正：

- 过度心理描写 → 删掉 2/3，只留最关键的一句
- 客气对话 → 给对话加"刺"，打断、抢话、说半句
- 万能形容词 → "深邃的目光"→ 用具体动作代替
- 总结式结尾 → 删掉，或改成悬念式断句
- 均衡描写 → 放大重点场景，压缩过渡
- 情感标签 → "他感到愤怒" → 用行为展示：攥拳/沉默/笑
- 解释性对话 → 用冲突和行动传递信息
- 过渡段 → 直接跳到下一个场景
- 修辞堆砌 → 一段最多一个修辞
- "不是……而是……"句式 → 直接删掉前半句，只保留结果
- 平衡感太强 → 让人更任性更极端

## ⚠️ 禁止句式

**强制禁止"不是……，而是……"句式。** 这是 AI 写作的高频特征，显得矫情和说教。直接说结果，不要先否定再肯定。

```
❌ 禁止：
   凌绝尘不是在炫耀，而是在展示剑道的极致。
   苏寒染不是害怕，而是不知道该说什么。
   他不是放弃了，而是选择了另一种方式。

✅ 正确：
   凌绝尘在展示剑道的极致。
   苏寒染不知道该说什么。
   他选择了另一种方式。
```

### 七、写作原则

1. **读者体验优先**：每句话都要让读者想点"下一行"
2. **节奏快于深度**：宁可牺牲文学性，不能牺牲节奏
3. **展示而非告知**：用动作、对话、感官代替直接说明
4. **冲突持续不断**：解决一个冲突的同时埋下下一个
5. **角色驱动情节**：角色做决定推动剧情
6. **克制留白**：关键处收住，不要把话说满
7. **风格统一**：同一版本内保持风格一致，不同版本间风格有明显差异
