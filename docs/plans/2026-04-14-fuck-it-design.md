# fuck-it 单章加戏 skill 设计

**背景**

现有 `vibe-noveling` 工作流里，`booming` 负责把后续剧情往更狠、更疯、更高冲突的方向炸开，但缺少一个只服务单章节内部强化的模式。

用户已经明确要求：

1. 新 skill 名称为 `fuck-it`。
2. 触发词以 `fuck it` 为主，并兼容 `fuck-it`。
3. 主要给 `novel-discuss` 用，同时也允许在 `novel-plan` 里直接调用。
4. 默认必须给出 3 套方案。
5. 这个 skill 只放大单章节内部的戏剧性、夸张感、冲突烈度、角色表现力和漫画感。
6. 绝对不能改变该章节原本的结束目标。
7. 不能只靠固定三种方向，必须内置 15 种单章加戏方向，并从中选择 3 种输出。

**目标**

新增 `plugins/vibe-noveling/skills/fuck-it/SKILL.md`，让它成为一个“同终点、强过程”的单章加戏器：

1. 读取最小必要上下文，先确认当前章的结束目标。
2. 先过一遍内置的 15 种单章加戏方向，再从中选出 3 套同终点但不同气质的强化方案。
3. 允许显著升级单章内的冲突等级、误会强度、公开场面、压迫感和表演感。
4. 强制把“终点不变”写成硬约束，防止加戏时偷偷改章末结果。
5. 如果当前在 `novel-discuss`，用户确认后把选中的方案交给 `novel-plan` 落正式大纲；如果当前就在 `novel-plan`，直接把选中的方案收束进当前大纲。

**设计决策**

### 1. 独立 skill，而不是塞进 `booming`

`booming` 的职责是把剧情线炸开，允许改写长期主线和后续结构；`fuck-it` 的职责恰好相反，它必须锁死本章终点，只改过程的戏剧烈度。两者边界不同，拆成独立 skill 更清晰。

### 2. 支持 `novel-discuss` 与 `novel-plan` 双入口

这个能力既可能出现在剧情讨论阶段，也可能出现在单章规划阶段。把它设计为独立 skill，可以被两个阶段共用，同时保持统一输出格式。

### 3. 默认三案，但三案来自 15 种方向库且共享同一章末目标

如果只给一案，很容易退化成普通建议；如果三案允许换终点，又会破坏这个 skill 的核心约束。因此默认三案，同时要求每一案都显式复述同一个“本章结束目标（固定）”。

同时不能把 skill 固化成“永远只给压迫升级 / 表演反杀 / 失控连锁”这三类。更合理的做法是内置 15 种单章加戏方向库，每次先过库，再根据当前章节筛出最适合的 3 种。

### 4. 放大的是章节内部张力，不是未来结构

输出要强调冲突升级链、角色表现力放大点、漫画感装置和局部代价，而不是去改写长期主线或整卷终局。需要大范围改命时，仍然应该切回 `booming`。

**影响范围**

- `plugins/vibe-noveling/skills/fuck-it/SKILL.md`
- `README.md`
- `.claude-plugin/marketplace.json`
- `tests/test_novel_write_workflow.py`

**验证**

通过静态文本契约测试确认：

1. `fuck-it` 触发词和双入口职责描述存在。
2. skill 明确承诺先过 15 种方向库，再给出 3 套同终点方案。
3. skill 明确禁止改变单章节结束目标。
4. skill 明确要求放大冲突链、角色表现力和漫画感装置。
5. README 明确写出 `fuck-it` 的定位、触发方式和与 `novel-plan` 的衔接。
