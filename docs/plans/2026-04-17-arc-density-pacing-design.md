# 弧段详略策略设计

> 日期：2026-04-17
> 状态：已实现

## 问题

当前写作流程中，20 个剧情点被平等对待——每个都是"写完就行"。但叙事需要详略得当：爽的地方拉长爽感，过桥的地方快速带过不废话。

## 设计决策

### 为什么从 discuss 开始，而不是 bookplan

- bookplan 是结构决策（beat 分配、卷职责段），不涉及内容密度
- discuss 是创意决策阶段，老板在讨论 arc 时会自然判断"这段对决要爽"、"那段转场别废话"
- 爽点经常横跨多章（蓄势→爆发→余波），只有 arc 级讨论才能看到全貌

### 三层传导

```
discuss arc 讨论      → 弧段详略策略（创意决策，AI 给建议，用户确认）
       ↓ 继承
novel-plan 场景块     → 场景级密度标记（继承章节密度，场景间可微调）
       ↓ 继承
novel-write 剧情点    → 写作约束（字数空间、内容范围、风格偏好）
```

### 三个密度档位

| 档位 | 含义 | 下游效果 |
|------|------|---------|
| 高（展开） | 爽点/高潮/转折 | 200-400字/点，必须含感官层次+角色反应+内心冲击 |
| 中（标准） | 常规推进 | 120-200字/点，标准处理 |
| 低（压缩） | 过桥/转场 | 50-100字/点，只保留动作+信息，砍环境和配角心理 |

### 密度决策只做一次

- discuss 阶段确定 arc 级密度（哪些章节展开、哪些压缩）
- novel-plan 和 novel-write 都是继承+微调，不重新决策
- 不增加每章的规划负担

## 改动文件

| 文件 | 改动 |
|------|------|
| `skills/novel-discuss/SKILL.md` | 两段式收束后追加"弧段详略策略"步骤，AI 主动给建议 |
| `skills/novel-plan/SKILL.md` | 读 arc 时继承密度，节奏定位展示密度，场景块加密度和字段 |
| `skills/novel-plan/references/output.md` | 大纲模板基本信息加密度和字段，场景块加密度和字段 |
| `skills/novel-write/SKILL.md` | 剧情点继承密度，写作约束追加详略约束 |
| `skills/novel-write/references/merge-rules.md` | 新增密度偏好规则（高密度偏好展开型风格，低密度偏好压缩型风格） |

# 小说宪法设计

> 日期：2026-04-17
> 状态：已实现

## 问题

每本小说有不同的创作灵魂，但当前流程只有一套通用的"默认创作风格基线"。缺少一个**高度抽象、跨所有阶段生效、可迭代**的最高创作准则。

## 设计决策

### 为什么放在 CLAUDE.md

- CLAUDE.md 是每个 session 自动加载的，宪法天然"始终在场"
- 现有的"默认创作风格基线"本来就在 CLAUDE.md，宪法是它的升级版
- 所有技能已经读 CLAUDE.md，不需要加新读取链路

### 宪法结构

| 版块 | 作用 |
|------|------|
| 灵魂 | 一句话定义核心体验 |
| 爽感公式 | 爽感来源和递进节奏 |
| 叙事纹理 | 节奏、文字质感、信息密度 |
| 主角光谱 | 主角吸引力和与读者关系 |
| 作者人格 | 写作时的 roleplay 人格（六维度） |
| 绝对红线 | 不可能出现的东西 |
| 风格锚点 | 参考作品定位 |
| 默认规则 | 宪法未覆盖时的退回层 |

### 作者人格六维度

参考 [Persona Engineering 101](https://medium.com/@christina.kennedy/persona-engineering-101-writing-with-intentional-voices-be27b07efb34) 的 PERSONA 框架，为小说写作场景定制：

| 维度 | 含义 | 写作影响 |
|------|------|---------|
| 叙事姿态 | 叙述者与读者的关系 | 是否允许旁白态度、叙述者评论 |
| 价值底色 | 对权力、正义、人性的根本态度 | 决定角色命运走向 |
| 情感温度 | 叙事的情绪基调 | 是否煽情、用感叹号、堆情绪 |
| 幽默方式 | 幽默的类型和尺度 | 是否允许吐槽、段子、POV外搞笑 |
| 信息习惯 | 信息揭示的策略 | 藏到爆发 / 直给 / 铺垫后揭示 |
| 叙事癖好 | 作者独有的标志性手法 | 动作代替心理 / 反应代替直述等 |

### 默认人格

初始值为赛博精神病风格量身定做：
- 叙事姿态：疯狂的叙述者，像嗑了药的赛博精神病
- 价值底色：世界是个疯人院，暴力是通用语言，混乱才是秩序
- 情感温度：极端两极——冰到骨子里的冷笑，或爆到天灵盖的癫狂
- 幽默方式：黑色幽默、荒诞、血腥搞笑
- 信息习惯：脑洞轰炸，离谱的信息当正常甩
- 叙事癖好：把暴力写成艺术，把疯狂写成浪漫

### 生命周期

```
novel-init        → 创建宪法（含默认人格）
novel-discuss     → 讨论收束后检查是否触发宪法修正，有则更新 CLAUDE.md
novel-plan        → 读 CLAUDE.md，宪法优先于默认规则
novel-write       → 读 CLAUDE.md，合并时向作者人格收束
```
