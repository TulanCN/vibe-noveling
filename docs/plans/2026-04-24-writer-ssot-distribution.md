# writer agent 章内 SSoT 分布调度设计

## 背景

当前 `novel-write` 已经要求 writer subagent 在正文阶段把一章细切为 20 个剧情点，并按固定标题落文。

原先的 SSoT 约束主要停留在“用随机字符串片段驱动首句 / 意象 / 转场 / 台词节奏”的局部随机层面。它能减少完全一致的重复，但仍容易出现以下问题：

- 相邻多个剧情点虽然“随机”了，但主导戏剧招法依然相似
- writer 会顺手反复使用自己最熟悉的一种推进方式
- 随机性只作用于单点，不作用于整章分布
- 最终效果更像“局部打散”，不像“整章内部有真实变化”

用户希望把负责正文的 Agent 改成更接近 Sakana AI `String Seed of Thought` 原文里“distribution-faithful and diverse generation”的思路：

- 不是单次随机挑一个模板
- 而是先用种子约束一整批生成对象的分布
- 让不同生成样本之间既有差异，又不失控偏向某一个惯用模式

## 目标

把 writer subagent 的 SSoT 从“局部随机器”升级为“章节级分布调度器”，降低 20 个剧情点在以下维度上的趋同：

- 首句起手
- 主导戏剧手法
- 转场方式
- 台词组织方式

## 设计原则

1. **整章唯一种子**：每个 writer 在本章只生成 1 颗 SSoT 种子，供整章调度使用
2. **先规划后落文**：先完成 20 个剧情点的分桶、配额、槽位分配，再逐点写作
3. **限制邻近重复**：相邻剧情点和同场景簇内禁止机械复用同一招法
4. **保持风格人格稳定**：SSoT 不改变肘子 / 大仲马的人格，只调度它们在章内使用不同招法的次序与配比
5. **允许一次局部替换**：当某个槽位导致语句明显不顺时，可在同层相邻候选中替换一次，但不能整体弃用分布规划

## 落地改动

### `plugins/vibe-noveling/skills/novel-write/SKILL.md`

将“SSoT 创作多样性约束”从单点随机升级为章节级分布规则：

- 增加“先分桶，再写作”
- 增加“先配额，再落点”
- 增加“分布忠实，不追求平均”
- 把执行层拆成四层：首句起手 / 戏剧手法 / 转场与收束 / 台词组织
- 为四层各自设置相邻禁重与章节覆盖约束

### `plugins/vibe-noveling/agents/writer-zhouzi.md`
### `plugins/vibe-noveling/agents/writer-dazhongma.md`

新增统一的“SSoT 章内防趋同执行规约”，确保 writer agent 本体也知道：

- 要先做章节级分布规划
- 要按四个维度做槽位分配
- SSoT 是风格内部调度器，不是风格替代器

### `README.md`

同步公开行为说明，让外部描述与 prompt contract 一致。

### `tests/test_novel_write_workflow.py`

新增最小回归测试，锁定：

- README 已公开声明 writer 的章内 SSoT 分布规划
- `novel-write` 已声明章节级 SSoT 分布约束
- 两个 writer agent 已声明同一套防趋同执行规约

## 预期收益

- 降低“每个剧情点都用同一种起手和推进方式”的模板感
- 让同一风格在章内也有层次变化，而不是只在风格之间有区别
- 把 SSoT 从“随机点缀”推进到“受控分布”，更接近原论文思路

## 非目标

- 不改变 20 个剧情点结构
- 不改变肘子 / 大仲马的风格定位
- 不引入新的 writer agent
- 不把 SSoT 计算过程暴露给用户
